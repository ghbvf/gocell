# slowgate allowlist — tests permitted to exceed the default 5s budget.
#
# Format: each data line is `Package<TAB>TestName`. Every data line MUST
# be preceded by a `# <reason>` comment explaining why the wall-clock cost
# is unavoidable. The companion archtest SLOWGATE-ALLOWLIST-01 enforces
# both the comment rule and that every (Package, TestName) maps to a real
# top-level test func. Independent table-driven negative cases for the
# parser live in TestLoadSlowgateAllowlist_ParseRules (same file).
#
# Threshold rationale (5s, see tools/slowgate/main.go defaultThreshold):
# the dominant >2s test population in this repo is packages.Load-bound
# and subprocess-go-toolchain — both swing 2–7s from cold cache + GHA
# runner CPU contention. A 2s threshold produces a long tail of allowlist
# churn that is pure noise; 5s preserves real signal (a sleep regression
# from 100ms to 6s is still caught) without being trip-wire on CI flakes.
#
# Adding entries: only when CI has empirically failed the gate. The
# failing slowgate step prints `SLOW <Package> <TestName> <Elapsed>` lines
# directly to stderr in the GHA step log — append the matching pair here
# under the appropriate `# <category>:` comment with a non-empty reason.
