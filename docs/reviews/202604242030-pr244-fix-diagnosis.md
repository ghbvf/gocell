### F1 — 多 pod 拓扑门禁缺失（P1 分歧）

**状态：** CONFIRMED
**位置：** `cmd/corebundle/shared_deps.go:132-141` + `runtime/bootstrap/topology.go`
**调用链：**
```
LoadSharedDepsFromEnv
  → internalGuardFromEnv(adapterMode) [controlplane.go:66]
      → auth.NewInMemoryNonceStore(5m + 30s) [controlplane.go:86]  ← 总是 in_memory
  → SharedDeps.Validate() [shared_deps.go:100]
      → validateControlPlane() [shared_deps.go:127]
          → d.InternalGuard.NonceStore().Kind() [shared_deps.go:137]
          → if kind == NonceStoreKindNoop → error    ← 只拒 Noop
          → NonceStoreKindInMemory 静默通过
```

**根因：**
- 代码层面：`validateControlPlane` 只拿 Kind 判 Noop，不结合拓扑信息。
- 架构层面：`bootstrap.Topology` 无 `ReplicaCount` / `SinglePod` 字段；startup 校验无法知晓部署拓扑，被迫信任运维纪律。
- 历史层面：Kubernetes kube-apiserver 把多实例安全状态（lease/certs）靠外部共享存储，无法单实例 fallback 悄悄上线；本 PR 选了"默认 in_memory + 文档警告"路线，结构性弱一档。

**复杂度：Cx2**（涉及 shared_deps.go + controlplane.go + 可能新增 Topology 字段 + env-vars.md，跨 2-3 文件）

**方案 A（最小）：warn-only**
```go
// controlplane.go, internalGuardFromEnv 末尾
if store.Kind() == auth.NonceStoreKindInMemory {
    slog.Warn("nonce store is in-memory; replay protection is single-pod only — inject a distributed store before horizontal scaling",
        slog.String("env_var", "GOCELL_SERVICE_SECRET"))
}
```
改动：`controlplane.go` +3 行。不引入新 env。
缺点：warn 可被日志聚合淹没，不会阻止运维误扩容。

**方案 B（彻底）：显式拓扑声明**
```go
// bootstrap/topology.go 新增字段
type Topology struct {
    AdapterMode string
    SinglePodReplayProtection bool  // 从 GOCELL_SINGLE_POD=1 读
    ...
}

// shared_deps.go validateControlPlane
} else if kind := d.InternalGuard.NonceStore().Kind(); kind == auth.NonceStoreKindNoop {
    errs = append(errs, noop_err)
} else if kind == auth.NonceStoreKindInMemory && !d.Topology.SinglePodReplayProtection {
    errs = append(errs, errcode.New(errcode.ErrControlplaneNonceStoreMissing,
        "in-memory nonce store requires GOCELL_SINGLE_POD=1 to acknowledge single-pod scope; multi-pod deployments must inject a shared store via WithServiceTokenNonceStore"))
}
```
改动：`topology.go` +4 行 + `shared_deps.go` +3 行 + `controlplane_guard_test.go` 补用例 + `env-vars.md` 新条目。
优点：fail-fast 默认阻止多 pod 静默不安全上线。
缺点：要求运维在单 pod 场景显式声明，增加配置复杂度。

**推荐：** 方案 B。理由：本 PR 目标明确是"将 replay 防御做成结构性不变量"（PR 描述原文），warn-only 的结构强度与 PR 目标语气不匹配；`GOCELL_SINGLE_POD=1` 是一次性部署声明，运维成本低于一次静默安全漏洞的代价。但此决策会调整 runtime/bootstrap 公共签名，属于架构级决策，**需产品/架构席位书面 confirm 后执行**。

### F4 — TTL 边界测试未推进时钟（P2 测试精度）

**状态：** CONFIRMED
**位置：** `cmd/corebundle/controlplane_guard_test.go:166-181`

当前代码：
```go
func TestInternalGuardFromEnv_NonceStoreTTLExtendsBeyondTokenWindow(t *testing.T) {
    ...
    store := guard.NonceStore()
    require.NoError(t, store.CheckAndMark(ctx, "nonce-ttl-probe"), "first insert must succeed")
    require.ErrorIs(t, store.CheckAndMark(ctx, "nonce-ttl-probe"), auth.ErrNonceReused,
        "same nonce within TTL must be rejected")
}
```

**根因：** 测试名"extends beyond token window"暗示验证"nonce 在 5 分钟 token 边界后仍被保留"，但断言只验证了"立即重放"，时钟未推进。任何 TTL < 1 秒的实现也能通过。

**复杂度：Cx1**（1 文件，新增 ~20 行或改写既有 case）

**方案：** 改用 `WithNonceClock` 构造可控时钟，但 `internalGuardFromEnv` 内部不暴露 clock 注入。两种实现：

**方案 A（测试内直接构造 store）：**
```go
func TestInternalGuardFromEnv_NonceStoreTTLExtendsBeyondTokenWindow(t *testing.T) {
    now := time.Unix(1700000000, 0)
    store, err := auth.NewInMemoryNonceStore(
        auth.ServiceTokenMaxAge+30*time.Second,
        auth.WithNonceClock(func() time.Time { return now }),
    )
    require.NoError(t, err)

    require.NoError(t, store.CheckAndMark(t.Context(), "nonce-a"))

    // 临界 token（age = ServiceTokenMaxAge - 1s）依然在 store 保留窗口内
    now = now.Add(auth.ServiceTokenMaxAge - time.Second)
    require.ErrorIs(t, store.CheckAndMark(t.Context(), "nonce-a"), auth.ErrNonceReused,
        "nonce must still be protected at ServiceTokenMaxAge-1s")

    // 推进过 buffer 后允许重用（证明 TTL 不是无限长）
    now = now.Add(31 * time.Second)
    require.NoError(t, store.CheckAndMark(t.Context(), "nonce-a"),
        "nonce must be reusable after TTL window")
}
```
改动：controlplane_guard_test.go ~25 行。不依赖 internalGuardFromEnv（因为后者用 time.Now）。

**方案 B（暴露内部 clock 注入）：** 给 `internalGuardFromEnv` 加 option 或测试钩子。侵入性大，不推荐。

**推荐方案 A。** 测试语义更明确：验证的是 `NewInMemoryNonceStore` 在 `ServiceTokenMaxAge+30s` TTL 下的边界行为，而非 `internalGuardFromEnv` 的装配。若需再加一个"装配后 TTL 正确"的测试，用 F6 的 MaxAge() 访问器做一行断言即可。

**时机：** 现在做。

### F8 — nonce map 容量治理（P2 Backlog）

**状态：** CONFIRMED
**位置：** `runtime/auth/nonce.go:127-136`

```go
if len(s.seen) >= s.maxEntries {
    for k, exp := range s.seen {
        if now.After(exp) {
            delete(s.seen, k)
        }
    }
}
// 清理后若仍超限，照样插入，map 无上限膨胀
```

**根因：**
- 场景：高并发持续 ~100k nonce/5min 以内（即大部分未过期），lazy prune 每次调用都空跑，map 持续膨胀。
- 安全代价：若攻击者能以高速率填充 nonce，内存是可控压力点（非正确性漏洞）。
- 产品代价：无指标能告警"nonce store 接近上限"。

**复杂度：Cx2**（nonce.go + metrics.go + 测试）

**方案要点：**
1. 清理后若 `len(seen) >= maxEntries` → 返回 store error（引导请求失败 500 或 429）。
2. 新增指标 `gocell_auth_nonce_store_size{store="in_memory"}`。
3. 可选：LRU eviction 替换 map（但 LRU 会违反 TTL 语义——可能提前驱逐未过期 nonce，破坏 replay 防御）。

**归属判定：RELATED，不是 PR-A25 目标。** PR 描述目标是"关闭 replay 窗口 + 结构性不变量"，不包含容量治理。登记 backlog follow-up。

**backlog 条目：**
```markdown
- [ ] **P2 auth-nonce-capacity**：InMemoryNonceStore 高基数 nonce 压力治理。
      - maxEntries 超限时拒绝入站（返回 store error → 500）
      - 新增 gocell_auth_nonce_store_size 指标
      - 评估 LRU eviction vs TTL 语义冲突
```

### F10 — env-vars.md raw bytes 文档说明（P2）

**状态：** CONFIRMED 轻微
**位置：** `docs/ops/env-vars.md` `GOCELL_SERVICE_SECRET` 条目

现有文本：
> Value is used as raw UTF-8 bytes (not base64-decoded). To generate: `openssl rand -base64 32`. ...

Round-2 指出：`openssl rand -base64 32` 生成 44-char 输出（base64 编码 32 字节 = 44 字符，含 padding `=`），当作 raw UTF-8 使用时长度 44 字节 ≥ 32-byte minimum，功能上没问题。但混合"base64 生成命令 + 当 raw 用"的说明可能误导运维以为需要 decode。

**复杂度：Cx1**（1 文件，+2 行）

**方案：** 文档追加 hex 备选 + 长度解释
```markdown
Value is used as raw UTF-8 bytes (not base64-decoded); any UTF-8 string of
≥ 32 bytes is acceptable. Recommended generators:
- `openssl rand -base64 32` → 44 printable chars (base64 padded), used as raw bytes
- `openssl rand -hex 32`    → 64 hex chars, used as raw bytes
Both meet the 32-byte minimum; choose whichever your secret manager accepts.
```
