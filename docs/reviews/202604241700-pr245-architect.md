## 总体结论: **APPROVE (with Cx3 nits)**

PR-A5c 完成了 3 个 L2 Cell 出站事件接口的统一：
- 引入 `WithEmitter` / `WithOutboxDeps` 两条互斥路径，取代原先分散的 `WithPublisher`/`WithOutboxWriter` 裸暴露；
- 在 `kernel/cell` 新增 `ResolveEmitter` + `DirectPublishModeForDurability` 统一仲裁；
- 新增 `kernel/outbox.DurabilityReporter` 可选扩展接口；
- 新增 archtest `OUTBOX-CELL-01` 锁定 Cell 边界。

分层依赖正确，kernel 无反向依赖，Cell 无 adapters 直连。`runtime/outbox/envelope.go` 删除后 `kernel/outbox.{Marshal,Unmarshal}Envelope` 单一权威化，消除了双实现风险。互斥语义在 Init 做 fail-fast；DirectPublish 策略由 kernel 集中，配合了 ref 注释（client-go / watermill / fx / go-zero）。

仅有少量 Cx3 级文档与职责边界可优化，均不阻塞合并。

## 分层与接口稳定性核对

1. **kernel/ → runtime/,adapters/,cells/ 依赖**：`kernel/outbox/emitter.go` 仅 import `context`/`log/slog`/`reflect`/`fmt` + `pkg/errcode`；`kernel/cell/mode_resolver.go` 仅 import `kernel/outbox`/`kernel/persistence`/`pkg/errcode` + `log/slog`。**零违规**。
2. **cells/ → adapters/**：三 Cell 的 `cell.go`/`cell_init.go` 均只 import `kernel/*` + `runtime/*` + `pkg/*` +（configcore）`jackc/pgx/v5/pgxpool` 与 `prometheus/client_golang`（通过 runtime/crypto 间接，但 pgxpool 是 configcore 直接 depends — 这在旧 PR 已 grandfathered，不属本 PR 新增违规）。**无本 PR 新增违规**。
3. **runtime/ → kernel/**：`runtime/outbox/relay.go` import `kernel/outbox` 作为 kout，`runtime/eventbus/eventbus.go` 同模式；**正确**。`adapters/rabbitmq/subscriber.go` import `kernel/outbox` 实现 Subscriber 接口；**正确**。
4. **接口稳定性**：`outbox.Emitter` 签名无变化；新增 `DurabilityReporter` 为 optional extension（不破坏既有 Emitter 实现）。`WithPublisher`/`WithOutboxWriter` 三 Cell 的删除属内部 API（GoCell 自用），符合 CLAUDE.md "重构不考虑向后兼容" 原则。**无 breaking change 对外**。
5. **互斥语义**：三 Cell Init 一致执行 `hasEmitter && hasPending → ErrCellInvalidConfig`；累积语义（nil 保留既有）对 `WithOutboxDeps(pub, nil)` + `WithOutboxDeps(nil, writer)` 分步调用友好，代价是同调用中传 nil 不会覆盖错填。**取舍合理**。
