### N1 [P2] [Cx1] `withContextMetadata` 缺显式 `value == ""` 短路

**文件**：`kernel/outbox/observability.go:115`
**问题**：当前 guard 是 `len(value) > MaxMetadataIDLen || !idutil.IsSafeID(value)`，依赖 `IsSafeID("")` 返回 false 来过滤空字符串。
**实证验证**：`pkg/idutil/id.go:32-34` 显示 `IsSafeID` 对空字符串返回 `false`：
```go
if len(s) == 0 { return false }
```
所以**当前代码功能上是正确的**——空字符串被 `!IsSafeID("")` 短路。但语义依赖 `idutil` 包的隐式契约，可读性差。
**修复建议**：在 `withContextMetadata` 第一行加 `if value == "" { return ctx }`，意图显式化。
**Cx**：1 个文件 1 行新增，Cx1。

### N3 [P2] [Cx1] FMT-19 AST 不解 `*ast.ParenExpr` 包装

**文件**：`kernel/governance/rules_wrapper.go:537`
**问题**：`vs.Values[0].(*ast.CompositeLit)` 在 `var x = (NoopTracer{})` 形式下会失败（因为 RHS 是 `*ast.ParenExpr`），从而误判为非-CompositeLit 违规（false positive）。
**实证验证**：`rules_wrapper.go:537-540` 代码确认无 `*ast.ParenExpr` 解包；测试矩阵无该 case。
**实际影响**：极低——`var x = (NoopTracer{})` 是非常罕见的 Go 风格，生产代码不会写。但治理规则应当对极少数合法形态准确，避免被合规吐槽。
**修复建议**：在类型断言前加解包：
```go
val := vs.Values[0]
for {
    if pe, ok := val.(*ast.ParenExpr); ok { val = pe.X; continue }
    break
}
cl, ok := val.(*ast.CompositeLit)
```
并加 `var x = (NoopTracer{})` 测试用例。
**Cx**：1 文件 + 1 测试用例，Cx1。

### N4 [P2] [Cx1] `slog.String("error", err.Error())` 应改 `slog.Any("error", err)`

**文件**：`adapters/postgres/outbox_store.go:254-256, 261-263`
**问题**：`scanClaimedEntry` 的两处 unmarshal warn 用了 `slog.String("error", err.Error())`，丢失 wrapped error 链信息。项目规范虽未硬性要求 `slog.Any`，但日志聚合工具按 attribute 类型解析时，`slog.Any` 能保留 `errors.Unwrap` 链；对调试 unmarshal 路径更友好。
**实证验证**：
```go
slog.Warn("outbox store: failed to unmarshal metadata",
    slog.String("entry_id", ce.ID),
    slog.String("error", err.Error()))   // ← 同一文件内两处
```
**修复建议**：两处改 `slog.Any("error", err)`，对齐项目其他模块（gauge：`grep -rn 'slog.Any("error"' kernel/ runtime/ adapters/ | wc -l` 远多于 `slog.String("error"`）。
**Cx**：2 行同文件改写，Cx1。

### N5 [P2] [Cx1] `traceParentFromContextIDs` 硬编码 sampled flags

**文件**：`kernel/outbox/observability.go:145`
**问题**：合成 traceparent 时硬写 `"-01"`（sampled=1）。原始 ctx 若来自 `traceFlags=00`（未采样）的上游 span，合成结果将错误地标记为 sampled，违反 W3C Trace Context 的 flags 透传约定。
**实证验证**：
```go
return "00-" + strings.ToLower(traceID) + "-" + strings.ToLower(spanID) + "-01"
```
ctxkeys 包当前没有 `TraceFlags*` 键，所以无法传递真实 flags。设计层面 GoCell 异步边界本就只携带 4 字段，没有保留 flags——这是有意识的取舍。
**修复建议**：在函数 godoc 中显式声明：
> TraceFlags is hardcoded to 0x01 (sampled) because GoCell's context model does not carry standalone trace-flags across async boundaries; all synthesized traceparents are seen as sampled by downstream consumers.

后续若要支持采样决策传播，再扩 ctxkeys + ObservabilityMetadata。
**Cx**：1 行注释，Cx1。

### N6 [P2] [Cx1] `stripMountPrefix` 死分支

**文件**：`runtime/auth/route.go:156-158`
**问题**：
```go
stripped := strings.TrimPrefix(fullPath, prefix)
if stripped == "" { return "/" }
if stripped[0] != '/' { return "/" + stripped }   // ← unreachable
return stripped
```
**实证验证**：`isPathSegmentPrefix` 已在调用前确保 `fullPath == prefix`（→ stripped=""，走第一个 return）或 `fullPath[len(prefix)] == '/'`（→ stripped 必以 '/' 开头）。第二个 if 永远 false。
**修复建议**：删除死分支并加防御性注释：
```go
stripped := strings.TrimPrefix(fullPath, prefix)
if stripped == "" {
    return "/"
}
// isPathSegmentPrefix guarantees fullPath[len(prefix)] == '/', so stripped
// always starts with '/' here.
return stripped
```
**Cx**：1 函数内删 2 行，Cx1。
