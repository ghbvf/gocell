## Finding 清单

### P0 — 阻塞合并

**[P0][Cx1][安全] `cells/accesscore/initialadmin/credfile_security_windows.go:91-93`**
`sddlContainsSID` 仅搜索 `";" + sidStr + ")"` 模式，未区分 ACE type (ALLOW `A` vs DENY `D`)。若攻击者把 ALLOW 替换为 DENY，`tampered=false` 仍通过，误判未被篡改。
建议: 搜索模式改为锁定 ALLOW ACE 前缀 `"(A;"` / `"A;;"`，或在遍历时同时检查 ACE type 字段。

### P1 — 应当修复

**[P1][Cx1][安全] `cells/accesscore/initialadmin/credfile_io.go:43-45`（TOCTOU）**
`os.Stat(path)` 跟随符号链接。在 `Stat` 返回 `ErrNotExist` 与 `SecureNewFile(tmpPath)` 之间，攻击者可把 `path.tmp` 替成指向特权文件的 symlink；Unix 侧 `os.OpenFile` 无 `O_NOFOLLOW`，会跟随链接写目标。PR 自述跳过 4 个 symlink 测试（V-A4），但 backlog 无具名 ticket，代码也无防护。
建议: Unix `SecureNewFile` 加 `syscall.O_NOFOLLOW`；或调用前 `os.Lstat`；并在 backlog 建 ticket。

**[P1][Cx1][安全] `cells/accesscore/initialadmin/credfile_security_windows.go:55` (PROTECTED bit 未校验)**
`VerifyOwnership` 仅校验 ACE count==3 + SID 出现，不校验 `PROTECTED_DACL_SECURITY_INFORMATION` 标志。攻击者取消 DACL 保护位、让父目录继承 ACE 进来维持数量 3，可绕过校验扩大权限。
建议: 在 `getFileDACL` 中检查 `SE_DACL_PROTECTED` control bit。

**[P1][Cx1][可维护性] `cells/accesscore/initialadmin/credfile_security_windows.go:96-108`（虚假约束）**
`containsSubstring` 注释称"to avoid importing strings in this build-tag-constrained file"。`strings` 是标准库，任何 build tag 下均可 import，是虚假约束；手写实现无 Rabin-Karp 优化。
建议: 删除 `containsSubstring`，直接 `strings.Contains`。

**[P1][Cx1][测试] `runtime/shutdown/shutdown_sigterm_unix_test.go:29-30`**
`time.Sleep(50ms)` 硬编码等待 `signal.Notify` 注册完成。在 `os-smoke` macOS 慢 runner 上可能不足，信号早于监听送达被丢弃 → 3s 后 flaky。
建议: 改用 channel 通知 `signal.Notify` 已注册，再 Kill。

**[P1][Cx1][运维/测试] `.github/workflows/_build-lint.yml:166-181`（os-smoke 无 vet/lint）**
`os-smoke` 仅 `go test`，Windows 特有条件编译错误（错误 build tag、未用变量）ubuntu lint 看不见，os-smoke 又无 lint。
建议: 增加 `go vet ./runtime/shutdown/... ./cells/accesscore/initialadmin/...`。

**[P1][Cx2][测试/安全] symlink 保护测试（V-A4）缺失**
PR 描述"跳过了 4 个 symlink 测试"，但 `credfile_symlink_*_test.go` 不存在（是"未写"非 `t.Skip`）。代码无防护也无测试，违反"不留小尾巴"。
建议: (a) 本 PR 补 `O_NOFOLLOW` + symlink 攻击测试；或 (b) backlog 具名 ticket + 代码 `TODO(V-A4)` 注释。
