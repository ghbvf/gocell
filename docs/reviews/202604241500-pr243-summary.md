## 合并阻塞 Finding（人工核验后保留）

### B1. Windows DACL 验证未区分 ALLOW / DENY ACE
- 文件: `cells/accesscore/initialadmin/credfile_security_windows.go:88-94`
- 代码证据:
  ```go
  func sddlContainsSID(sddl, sidStr string) bool {
      search := ";" + sidStr + ")"
      return len(sddl) > 0 && containsSubstring(sddl, search)
  }
  ```
- 风险: 若攻击者把 ALLOW ACE 改成 DENY ACE（`D;...;...;...;...;<SID>`），`sddlContainsSID` 仍返回 true，`VerifyOwnership` 将把被篡改的 DACL 当作未篡改；tamper-detection 失效。
- 修复: 搜索模式锁定 ALLOW 前缀，或显式解析每个 ACE 的 type 字段，只匹配 `A`（或 `AI`/`AO`）。

### B2. Unix `SecureNewFile` 无 `O_NOFOLLOW`（symlink TOCTOU）
- 文件: `cells/accesscore/initialadmin/credfile_security_unix.go:25-27`
- 代码证据:
  ```go
  func (u *unixCredfile) SecureNewFile(path string) (*os.File, error) {
      return os.OpenFile(path, os.O_WRONLY|os.O_CREATE|os.O_EXCL|os.O_TRUNC, 0o600)
  }
  ```
- 风险: `credfile_io.go:43-45` 用 `os.Stat` 跟随符号链接；`Stat` 与 `SecureNewFile(tmpPath)` 之间存在 TOCTOU 窗口。Unix 上攻击者把 `path.tmp` 替换成指向特权文件的 symlink，凭据将写入目标。PR 自述"跳过 4 个 symlink 测试（V-A4）"，但 backlog 无具名 ticket，代码无防护——"不留小尾巴"违规。
- 修复: Unix `SecureNewFile` 加 `syscall.O_NOFOLLOW`，或调用前 `os.Lstat` 拒绝 symlink；同时在 `docs/backlog.md` 登记 V-A4 后续 ticket。

### B3. Windows `VerifyOwnership` 未校验 `SE_DACL_PROTECTED`
- 文件: `cells/accesscore/initialadmin/credfile_security_windows.go:55-86, 141-156`
- 代码证据:
  ```go
  sd, err := windows.GetNamedSecurityInfo(
      path,
      windows.SE_FILE_OBJECT,
      windows.DACL_SECURITY_INFORMATION,
  )
  ...
  if dacl.AceCount != 3 { ... }
  ```
  `getFileDACL` 只请求 `DACL_SECURITY_INFORMATION`；`VerifyOwnership` 也未读取 `sd.Control()` 的 `SE_DACL_PROTECTED` bit。
- 风险: `applyRestrictedACL` 写入时用 `PROTECTED_DACL_SECURITY_INFORMATION` 阻断继承，但校验端不验证该保护位。若攻击者取消 PROTECTED、让父目录继承 ACE 进来并维持总数 3，校验仍通过，权限被悄悄扩大。
- 修复: `GetNamedSecurityInfo` 请求 control bits 或改用 `GetSecurityDescriptorControl`，显式要求 `SE_DACL_PROTECTED` 成立。

### B4. os-smoke 未设为 required check
- 文件: 需在 GitHub Settings 配置，不是代码问题
- 证据: `gh api repos/ghbvf/gocell/branches/develop/protection` 显示 `enforcement_level: off`
- 风险: PR 自述宣称 os-smoke 是 V-A2/V-A3/V-A4 的 CI gate，但 windows-latest 红灯时 PR 仍可合并
- 修复: 合并前在 GitHub 仓库 Settings 中把 `pr-ci / os-smoke (macos-latest)` 和 `pr-ci / os-smoke (windows-latest)` 列为 required check（check 名需与 Actions 产出完全一致）

### B5. `docs/operations/first-run-setup.md:35` 路径过期
- 内容: 文档引用 `cells/accesscore/internal/initialadmin/credfile.go::WriteCredentialFile`
- 实际: 路径已移出 `internal/`，文件被拆成 `credfile_io.go`，函数改为小写私有 `writeCredentialFile` / `formatPayload`
- 修复: 改为 `cells/accesscore/initialadmin/credfile_io.go::formatPayload`
