## 2. Findings

### Cx1（消费者即时可感知）

- **Cx1-1 [开发者体验]** `docs/operations/first-run-setup.md:166-226` 的完整 Bootstrap 演示仍用 Docker + `/run/gocell/initial_admin_password` 作为唯一流程，Windows 开发者若按 README 首屏 `go run ./examples/ssobff` 直跑本机，需跳到 §"Windows 开发" 小节。建议在 Step-2 加一个 "选择你的场景" 分支导航（Docker / macOS / Linux / Windows 四选一），减少页面扫描成本。
- **Cx1-2 [文档一致性]** `docs/operations/first-run-setup.md:244` 仍提示 `export GOCELL_STATE_DIR=$TMPDIR/gocell`，而 PR 已让 `ResolveBootstrapCredentialPath("")` 在 macOS 自动回退到 `~/Library/Application Support/gocell/run`——该 hint 对 "现代启动路径" 已是冗余（`maybeMacOSHint` 仅在用户**显式**写了 `/run/...` 时触发）。建议把 hint 降级为 "若你手动设成 `/run/...` 会触发…"，否则引导新用户误以为 macOS 必须显式 export。

### Cx2（结构性，影响后续使用者）

- **Cx2-1 [兼容性风险]** Linux 侧默认路径继续保持 `/run/gocell`，**但原 `examples/ssobff` 在 PR 前就是硬编码 `/run/gocell`**——该路径在非 systemd / 非 root 进程下本就不可写。PR 通过 `probeWriteable` 给出可操作错误信息，相比之前静默失败是净改进；但对"在 Linux 桌面 WSL/Docker Desktop 直跑 `go run`"的用户体验仍是 "启动失败 + 看 error hint 设 env"。建议在 `examples/ssobff/main.go` 启动 log 里把 resolved path **同时**输出 `mkdir` 结果 probe，或者在 Linux 非 root 场景 fallback 到 `os.UserConfigDir()`（与 macOS/Windows 对齐），和 systemd RuntimeDirectory 语义分离——这是首次运行体验（FRST）的真正痛点，且属于 Linux 原默认行为的回归保护。
- **Cx2-2 [验收标准缺失]** PR 未包含 V-A3 的 Windows 正向测试（只有 Unix 的 `shutdown_sigterm_unix_test.go`）。`os-smoke` CI 矩阵会跑 `./runtime/shutdown/...`，但缺一条 "Windows `os.Interrupt` 能被 `Wait()` 正确接收并触发 hook" 的 journey test——`signalsToWatch()` 分叉逻辑的 Windows 分支目前无 OS-specific 单测覆盖。建议补 `shutdown_interrupt_windows_test.go`（`//go:build windows`）。
- **Cx2-3 [开发者体验]** `docs/operations/first-run-setup.md:139-160` 的 Windows/macOS 小节没有 "如何读取凭据"的 **完整 curl 登录示例**（当前只示范 `Get-Content` / `cat`），新手 Windows 开发者难以把 PowerShell `$env:LOCALAPPDATA` 串到 Step 3 的 `curl ... -d "{\"password\":...}"`。Release note 建议外挂一段 end-to-end PowerShell 示例。

### Cx3（体系性，非本 PR 必修）

- **Cx3-1 [范围偏移]** 原 V-A2 finding 指出 `examples/ssobff/main.go:168` 用 `"/"` 拼接（V-A10 GOVERNANCE-WINDOWS-PATH-01）。**本 PR 未触碰 V-A10**——backlog 上 V-A10 仍处于 P1 待办。建议合并前在 PR 描述里明确标注 V-A10 的范围边界（"路径拼接只在 stateDir 场景修复"），避免后续 reviewer 误判 V-A10 已关闭。
- **Cx3-2 [文档链路]** `examples/iotdevice/main.go` / `examples/todoorder/main.go` 未使用 initial admin bootstrap，本 PR 对它们无影响；但 `iotdevice` 用 `demoAdminToken = "iotdevice-admin-demo-token"` 仍是硬编码，建议未来与 ssobff 的 bootstrap 能力统一（纳入 V-A19 或独立 ticket）。

## 3. Release Note 草稿

```markdown
### Cross-platform initial admin bootstrap

- `examples/ssobff` now runs out-of-the-box on macOS and Windows. The initial
  admin credential file location follows per-OS defaults
  (`/run/gocell/` on Linux, `~/Library/Application Support/gocell/run/` on macOS,
  `%LOCALAPPDATA%\gocell\run\` on Windows). Override via `GOCELL_STATE_DIR`.
- `runtime/shutdown` now correctly registers `os.Interrupt` only on Windows
  (SIGTERM is not deliverable there); Unix keeps SIGINT+SIGTERM behaviour.
- Symlink-based tests (`runtime/config/watcher_test.go`,
  `kernel/governance/validate_test.go`) now skip on Windows runners lacking
  `SeCreateSymbolicLinkPrivilege`; behaviour is unchanged on Unix.
```

无 breaking 变更：Linux `/run/gocell` 默认保持不变，env `GOCELL_STATE_DIR` 语义稳定。
