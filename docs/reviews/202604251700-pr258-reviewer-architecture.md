# PR #258 Architecture Review — `526-pr-a14b-route-group`

Base: `origin/develop`
Reviewer: architect role
Date: 2026-04-25

Scope: three-listener physical separation + typed `ListenerRef` + `Policy`
interface with 6 implementations + `RouteGroupContributor` declarative API +
`HTTPRegistrar` deletion + PR-A35 absorption.

## Executive Summary

Architecture is sound. Layer purity holds (kernel/ stays on stdlib-only;
runtime/bootstrap does not reach into cells/ or adapters/). The
`Policy` interface + `mountablePolicy` split is a well-executed cross-package
structural-typing pattern. Main concerns are: (1) `cell.Policy` being too
minimal to enforce the `Apply` contract at compile time (silent-skip in
`PolicyStack`); (2) ergonomic overlap between listener-default Policy and
`WithAuthMiddleware`/`WithAuthDiscovery`; (3) `cell.ListenerRef` is a
deployment concept leaking into the cell-layer contract surface;
(4) `PolicyVerboseToken` semantics vs. a mux-wide policy mismatch; and
(5) three distinct auth mechanisms (listener policy / `auth.Declare` flags /
route-level `RouteDecl.Policy`) without a single documented precedence table.

## Findings

| ID | Tier | Scope | Title | Rationale | Proposed fix |
|----|------|-------|-------|-----------|--------------|
| ARCH-01 | C1 | IN_SCOPE | `cell.Policy` contract is too weak — `PolicyStack` silently drops non-`mountablePolicy` members | `kernel/cell/policy.go:12` declares only `Describe() string`. `bootstrap.PolicyStack` at `runtime/bootstrap/policy.go:62-70` accepts `cell.Policy` but filters to `mountablePolicy` via type assertion, silently dropping any `cell.Policy` impl that does not live in `runtime/bootstrap`. That is an open door for "my custom Policy compiles, builds, and runs — but installs zero middleware." Per `AGENTS.md`/`CLAUDE.md` —空实现/no-op/fallback 必须写明业务原因；静默降级 is explicitly disallowed | Either (a) make `cell.Policy` require an `Apply(interface{ ... })` method that is neutral of `*chi.Mux` (e.g. accept `cell.RouteMux`-shaped target) so kernel actually owns the contract; or (b) keep the current split but fail-fast in `PolicyStack` when a member does not implement the runtime extension (return `error`, don't silently skip). The godoc at `runtime/bootstrap/policy.go:56-61` acknowledges the silent-drop — acknowledgement is not an excuse. |
| ARCH-02 | C2 | IN_SCOPE | `cell.ListenerRef` embeds deployment topology into the reusable cell contract | `cells/accesscore/cell_routes.go:25,93` hardcodes `cell.PrimaryListener` and `cell.InternalListener`. These constants are defined in `kernel/cell/listener.go:28-33`. A reusable cell shipped to a different assembly with different listener topology (e.g. "edge+admin+observability+internal" four-tier or "single public" single-tier) cannot redirect its routes without code edits. CLAUDE.md positions cells as "cross-assembly reusable" — baking a 3-listener topology into kernel vars violates that | Introduce indirection: either (a) cells declare semantic intent (`cell.BusinessPublic` / `cell.ControlPlane` / `cell.Infra`) and assembly.yaml maps semantic names → physical ListenerRef; or (b) cells receive the `ListenerRef` as a parameter via `RouteGroupContributor` receiving a `ListenerResolver` rather than hardcoding the var. For current scope, at minimum document in kernel/cell/listener.go that these three are convention-not-contract and assemblies are expected to extend the set. |
| ARCH-03 | C2 | IN_SCOPE | Three auth mechanisms lack a documented precedence | (1) Listener-default Policy from `WithListener(..., defaultPolicy, ...)` installs mux-level middleware in `router.buildMux` (`runtime/http/router/router.go:368-370`). (2) `auth.Declare` with `Public/PasswordResetExempt/Delegated` flags feeds `FinalizeAuth` matchers. (3) `RouteDecl.Policy` per-route wraps handler via `RequirePolicy` (`runtime/auth/routedecl.go:72-75`). These three layer at different middleware levels. A cell author cannot determine from one source "which policy wins when X conflicts with Y" — e.g. if listener policy = `PolicyJWT` and route has `Public:true`, the public matcher bypasses JWT; but if listener policy = `PolicyServiceToken` + route has `Public:true`, service token still runs (it's not aware of the auth public matcher) | Publish a precedence + composition matrix in `docs/ops/listener-topology.md` (or `docs/architecture/auth-layering.md`): matrix of (listener policy, auth.Declare flag, RouteDecl.Policy) triple → effective behaviour. Add a runtime startup log line summarising the effective chain per declared route. |
| ARCH-04 | C2 | IN_SCOPE | `PolicyVerboseToken` applied mux-wide is semantic overkill | `runtime/bootstrap/policy_verbose_token.go:50-67` gates any request carrying `?verbose`. As a listener-default Policy it runs for every route on that listener. But the original PR-A35 intent (absorbed) is strictly about `/readyz?verbose` — not an arbitrary route with a `verbose` query param. A cell that happens to accept `GET /foo?verbose=true` as a legitimate query (pagination verbose mode, logs verbose mode, etc.) mounted on the HealthListener would now be gated unexpectedly. The CLAUDE.md "空实现/no-op/fallback 必须写明业务原因" suggests surprising semantics need documentation too | Narrow scope: either (a) rename to `PolicyReadyzVerboseToken` and scope the Apply to `r.URL.Path == "/readyz"` only; or (b) keep name but restrict via path-prefix option. Current implementation at file-level is architecturally a middleware, not a "policy" — listener policies should guard the transport, not leak into application query semantics. |
| ARCH-05 | C2 | IN_SCOPE | `WithAuthMiddleware` / `WithAuthDiscovery` vs. `PolicyJWT` are two paths to the same thing | `runtime/bootstrap/bootstrap_phases.go:680-690` wires `router.WithAuthMiddleware(b.authVerifier)` onto the primary router when `b.authVerifier != nil`. Simultaneously, the operator can pass `PolicyJWT(verifier)` as the defaultPolicy argument to `WithListener(cell.PrimaryListener, addr, PolicyJWT(v))`, which installs JWT middleware via `policyJWT.Apply` (`runtime/bootstrap/policy_jwt.go:16-18`). Both code paths install `auth.AuthMiddleware`. If a caller accidentally does both, JWT runs twice. There is no phase0 validator rejecting that combination | Either (a) make it a compile error by requiring callers to choose one — if `b.authVerifier != nil` AND any listener's policy is `*policyJWT` → phase0 error; or (b) fully deprecate `WithAuthMiddleware`/`WithAuthDiscovery` in favour of the policy path (single way). `WithAuthDiscovery` still adds value (auto-finds verifier from cells) but should then feed into `PolicyJWT` automatically rather than into a parallel wiring lane. |
| ARCH-06 | C3 | IN_SCOPE | Phase0 does not require PrimaryListener; rigid "at least one" is too loose | `runtime/bootstrap/bootstrap.go:331-341` `validateHTTPListenerConfigs` requires only `len(b.listenerConfigs) > 0`. The commented "pure worker service" case in the review brief is worth addressing: if the caller declares only a HealthListener (for k8s probes) + no PrimaryListener and no cell contributes routes, the framework still constructs a HealthListener via `phase5CollectRouteGroups` → `resolveHealthListenerRef` (falls through to `cell.HealthListener`). That path works but is fragile: `phase5FinalizePrimaryRouter` short-circuits on no primary (`:622-626`) — fine — but the public-prefix handling (`:680-689`) is primary-only, so some invariants (internal-prefix isolation) silently disappear with no log | Allow "health-only" deployments explicitly by documenting the supported shapes (primary-only / primary+internal / primary+internal+health / health-only / internal-only) in `docs/ops/listener-topology.md`. Add a phase0 validation that warns (or errors) when RouteGroupContributors declare a listener the assembly did not configure (already done at `phase5MountRouteGroups:603-607`, good). For health-only, log an explicit "running in non-primary topology" at startup. |
| ARCH-07 | C3 | IN_SCOPE | `HealthRouteGroups` auto-injection happens unconditionally | `runtime/bootstrap/bootstrap_phases.go:588-598` `remapHealthGroups` always emits 2-3 health groups (healthz/readyz + optional metrics). If a team wants to front-load health via a sidecar (istio-proxy, k8s-native handler) they cannot opt out. The `HealthRouteGroups(h, nil)` call is always invoked. No `WithoutHealthRoutes()` escape hatch | Add `WithoutHealthRoutes()` option that suppresses the auto-injected health groups; useful for sidecar-probed deployments or pure worker processes. Low-cost addition, high-value for assembly flexibility. |
| ARCH-08 | C3 | IN_SCOPE | `cell.Policy` / `runtime/bootstrap.mountablePolicy` duck-typing on exported `Apply` creates hidden coupling across packages | `runtime/http/router/router.go:393-400` `applyPolicyToMux` defines its own anonymous `mountable interface { Apply(mux *chi.Mux) }` and uses type assertion to satisfy it from sibling package policies. This works (structural typing) but the `*chi.Mux` parameter hardcodes chi into every Policy impl. If GoCell ever swaps chi, every Policy's `Apply` signature breaks | Define a narrow `PolicyTarget` abstraction in `kernel/cell` that `Apply(PolicyTarget)` accepts — hide chi.Mux behind `RouteMux`-equivalent. Then kernel's Policy interface can state `Apply(target)` directly, removing the cross-package duck-typing entirely and letting `PolicyStack` enforce compile-time conformance. |
| ARCH-09 | C3 | IN_SCOPE | `bootstrap_phases.go` phase5 decomposition is reasonable but `buildMux` in router is borderline | `phase5BuildRouters` itself is clean (6 helper calls, :502-522). `router.buildMux` at `runtime/http/router/router.go:335-383` branches on tracer/metrics/policy/rate/circuit/auth within a single function — ~8 conditionals, still under 15 cognitive but trending. New per-listener logic means this function now runs N times instead of once | Not blocking. Monitor; if another middleware lands here, extract `applyProtectionChain` + `applyObservabilityChain` helpers before going over 15. |
| ARCH-10 | C3 | IN_SCOPE | Framework-comparison citations match code but `k8s apiserver` claim is partial | PR cites "ref: kubernetes/apiserver pkg/server/secure_serving.go — pre-bind listener" at `runtime/bootstrap/bootstrap_phases.go:989`. The pre-bind pattern is indeed adopted at `phase7BindListeners:1020-1044`. However, k8s apiserver also uses `genericapiserver.SecureServingInfo` to attach per-listener auth config, and the PR explicitly notes "rejected single-listener" (`runtime/http/router/router.go:272`) — consistent. uber-fx citation (`app.go Rollback`) is correctly reflected in `phase3InitAssembly:265-269`. go-kratos per-server middleware is reflected in `NewForListener` policy application. Cited match; no deviation | None — citations are accurate. |
| ARCH-11 | C3 | IN_SCOPE | LAYER-07 archtest exists and correctly forbids `cells/ → runtime/http/router`; LAYER-08 forbids `HTTPRegistrar` identifier | `tools/archtest/archtest_test.go:298-337` covers both. `kernel/cell/listener.go` / `policy.go` / `routegroup.go` import nothing beyond `kernel/cell`-internal types — verified clean. `runtime/bootstrap/*` imports only `kernel/*` + `runtime/*` + `pkg/*` + chi + std — no cells/ or adapters/ imports. Layer compliance is clean | None — archtest is well-placed. |
| ARCH-12 | C2 | IN_SCOPE | `WithPrimaryListener` / `WithInternalListener` / `WithHTTPPrimaryAddr` / `WithHTTPInternalAddr` shims contradict "no back-compat" policy | `runtime/bootstrap/bootstrap.go:344-375` keeps four deprecated shims. CLAUDE.md explicitly says "Review 和重构时不考虑向后兼容——当前只有 gocell 自身，没有外部调用方". These shims are dead weight that add an extra surface to maintain for no benefit | Delete the four `WithPrimaryListener` / `WithInternalListener` / `WithHTTPPrimaryAddr` / `WithHTTPInternalAddr` functions.
