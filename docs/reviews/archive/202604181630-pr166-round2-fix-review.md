## 1. 本次实际修复（合并去重后）

### F-MW-FALLBACK（P1，两轮 review 共识）

- **证据**: `runtime/auth/middleware.go` 原 `verifyAccessToken` 对 `verifier.(IntentTokenVerifier)` 做类型断言，失败时 fallback 到 `verifier.Verify(ctx, token)`
- **根因**: 安全不变量（access-only 才是安全默认）未类型化；`AuthMiddleware(TokenVerifier, ...)` 签名与语义矛盾，依赖运行时断言弥合
- **修复**:
  - `AuthMiddleware` 签名收紧为 `func(verifier IntentTokenVerifier, ...)`
  - 去掉 `verifyAccessToken` helper，直接 `verifier.VerifyIntent(ctx, token, TokenIntentAccess)`
  - 级联更新：`runtime/http/router/router.go::WithAuthMiddleware` / `runtime/bootstrap/bootstrap.go::{WithAuthMiddleware, authProvider}` 全部转 `IntentTokenVerifier`
  - `cells/access-core/cell.go::TokenVerifier()` 返回 `auth.IntentTokenVerifier`
  - `cells/access-core/slices/sessionvalidate/Service` 新增 `VerifyIntent` 方法（仅允许 `access` intent，其它 intent fail-closed）

### F-TEST-TYP-HELPER（P2，双维护）

- **证据**: `cells/access-core/slices/sessionvalidate/helpers_test.go::typHeaderForIntent` 是 `runtime/auth/jwt.go::jwtTypForIntent` 的副本
- **修复**:
  - `runtime/auth/jwt.go` 导出 `TypHeaderForIntent`（文档注明 test-only）
  - `helpers_test.go` 删重复实现，改用 `auth.TypHeaderForIntent`

### F-MW-FALLBACK-TEST（P2，回归）

- **修复**:
  - 新增 `TestAuthMiddleware_LegacyTokenVerifierIsCompileTimeRejected` 把"legacy TokenVerifier 不能传入"做成编译期断言
  - 清理 `intentMockVerifier` 死字段 `fallbackClaims/fallbackErr`；`Verify` 方法改为 sentinel error，防意外走到 fallback 路径
