### F-DX-001 [P1] [Cx1] tools/codegen/contractgen/doc.go:12
**问题**: `iface_gen.go` 注释为 "Service (HTTP) / Handler (event) interface (always)"，但实际 event 合同的 `iface_gen.go` 只含 `package <name>`，根本不生成 interface。

**证据**:
```go
// doc.go:12
//   - iface_gen.go    — Service (HTTP) / Handler (event) interface (always)
```
```go
// generated/contracts/event/audit/appended/v1/iface_gen.go:
package appended
```

iface.tmpl 中 event 分支无 type 声明。

**建议**: 改为 `// - iface_gen.go — Service interface (kind=http only; event generates empty package stub)`。

### F-DX-002 [P1] [Cx1] tools/codegen/contractgen/doc.go:11-13

//   - spec_gen.go         — ContractSpec var (kind=event only)

### F-DX-003 [P1] [Cx1] tools/archtest/cells_no_wrapper_contractspec_import_test.go:130-132
**问题**: `fset_new()` 注释称"避免遮蔽其他测试文件中的包级 fset"，但 `tools/archtest` 中根本没有包级 `fset` 变量。注释指向不存在的对象。

**证据**: Grep `var fset` 在 tools/archtest/ 无结果。

**建议**: 改注释为实际原因，或改函数名为 `newFileSet()` 并删多余注释（Go 命名惯例不应使用下划线）。

### F-DX-004 [P2] [Cx1] tools/codegen/contractgen/templates/*.tmpl
**问题**: 四个生成模板（handler/spec/subscription/iface）都没有顶部注释说明职责 / 修改注意 / 对应 archtest 守卫。对比 `kernel/wrapper/spec.go` 的高质量 godoc，模板文档密度差距明显。

**建议**: 在每个 .tmpl 顶部加 Go-style 块注释（`{{- /* ... */ -}}`），说明生成内容、对应 archtest gate ID、需同步的 golden 文件。

### F-DX-005 [P2] [Cx2] tools/codegen/contractgen/generator.go:199-288

**建议**: 抽取 `renderAllArtifacts(root, spec, pkgDir) ([]CodegenArtifact, error)`，两个对外函数复用。

### F-DX-006 [P2] [Cx1] tools/archtest/codegen_contract_gen_test.go:45-75
