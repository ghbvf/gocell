**[P1] [Cx2] [安全] `cmd/corebundle/shared_deps.go:137`**

问题: `validateControlPlane` 仅检测 `kind == auth.NonceStoreKindNoop`，对 `NonceStoreKindInMemory` 在多 pod 部署下的跨 pod replay 漏洞无任何运行时拦截。尽管文档已说明多 pod 须换 distributed 实现，但 startup 校验通过后没有任何机制阻止或警告运维人员以 in-memory 模式横向扩展——守门人已进门，后续扩容即造成静默跨 pod replay 窗口。


建议: 在 `slog.Info("controlplane guard installed", ...)` 后，若 `kind == NonceStoreKindInMemory` 追加一条 `slog.Warn("nonce store is in-memory; replay protection is single-pod only — inject a distributed store before horizontal scaling")`（Cx1 改动）。长期可考虑引入可选的 `GOCELL_REPLICA_COUNT` 环境变量：当 `> 1` 且 kind == InMemory 时升级为 startup 错误（Cx2，需人工决策）。

**[P1] [Cx1] [安全] `runtime/auth/nonce.go:109`**

问题: `NewInMemoryNonceStore` 接受任意正 `maxAge`（包括 1ns），但没有对 `maxAge < ServiceTokenMaxAge` 的最小值保护。调用方可能传入错误的较短 TTL，使得 replay 窗口在 token 有效期内重新出现。`internalGuardFromEnv` 目前正确传入 `ServiceTokenMaxAge + nonceStoreBuffer`，但如果将来有第二个调用方直接构造 store，该保护不存在。

证据:
```go
func NewInMemoryNonceStore(maxAge time.Duration, opts ...InMemoryNonceOption) (*InMemoryNonceStore, error) {
    if maxAge <= 0 {
        return nil, fmt.Errorf("auth: nonce store maxAge must be positive, got %v", maxAge)
    }
    // 无 maxAge < ServiceTokenMaxAge 检查
```

建议: 在构造函数中增加: `if maxAge < ServiceTokenMaxAge { return nil, fmt.Errorf("auth: nonce store maxAge %v is less than ServiceTokenMaxAge %v; a shorter TTL reintroduces the replay window", maxAge, ServiceTokenMaxAge) }`. 或至少加一条 `slog.Warn`。属于单文件改动，Cx1。

复杂度判定: 仅涉及 `runtime/auth/nonce.go` 一个文件，Cx1。

**[P2] [Cx1] [运维/部署] `docs/ops/env-vars.md`**

问题: `GOCELL_SERVICE_SECRET` 文档说明"To generate: `openssl rand -base64 32`"。但 `LoadHMACKeyRingFromEnv` 将 env value 当作 raw UTF-8 bytes 使用（非 base64 解码），注释也有说明。问题是 `openssl rand -base64 32` 的输出长度为 44 字节（包含 `=` 和换行符）——这超过 32 bytes，因此功能上无问题。但 `openssl rand -hex 32` 更直观（生成 64 hex chars = 64 bytes raw，也 >= 32 bytes）。混用建议可能让运维人员困惑"是否需要 base64 decode"。已有内联注释说明 no decode，但文档中可以更清楚地说明 raw bytes 语义。

建议: 文档补充一行说明："The value is used as raw UTF-8 bytes (not decoded). `openssl rand -base64 32` generates a 44-byte printable string; alternatively `openssl rand -hex 32` generates a 64-byte hex string. Both satisfy the 32-byte minimum."。Cx1，纯文档。
