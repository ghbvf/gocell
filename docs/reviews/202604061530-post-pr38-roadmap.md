## PR#42: RabbitMQ Solution B

来源: [202604061449-pr38-solution-b-report.md](202604061449-pr38-solution-b-report.md)

核心改造:
1. `outbox.Subscriber` handler 返回 `HandleResult{Disposition, Receipt}` 替代 `error`
2. `idempotency.Checker` 升级为 `Claim/Commit/Release` 两阶段模型
3. `ConsumerBase` 去掉应用侧 DLQ publish，改返回 Disposition
4. `Subscriber.processDelivery` 按 Disposition 决定 Ack/Nack，broker 处置成功后才 Commit/Release
5. 重连策略: `isRecoverableAMQPError` 覆盖完整 setup 生命周期 + backoff

测试清单:
- DLQ 失败后 release lease，redelivery 仍能再次执行业务
- setup 各阶段 transient/fatal error 表驱动测试
- Connection.Close() 后 Subscribe 不热循环
- broker-native DLX 集成测试
- EntryHandler/HandleResult/middleware 链接口测试

取舍: broker-native DLX 经典队列非强保证，如需更强终态保证需 quorum queue。
