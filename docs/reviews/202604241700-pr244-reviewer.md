## 六维度评估

### 1. 架构合规

`runtime/auth` 层新增 `Kind()` 方法扩展 `NonceStore` 接口——属于 breaking change，但 CLAUDE.md 声明无外部调用方，合规。`internalGuard` 作为 unexported 类型持有在 `SharedDeps.InternalGuard` 字段，限定在 `package main`，当前可接受；若 `cmd/corebundle` 未来拆包则需重构（见 OUT_OF_SCOPE 登记）。分层方向正确：`runtime/auth` 不依赖 `cmd/` 或 `cells/`，`cmd/corebundle` 依赖 `runtime/auth` 合规。

### 2. 安全/权限

real 模式防护完整。demo-key blocklist 在 current 和 previous secret 双路径都检查。nonce TTL = ServiceTokenMaxAge + 30s 覆盖边界充分。replay → 401 / store-infra-failure → 500 的分类逻辑正确。`ErrNonceReused` 作为 exported sentinel 使用裸 `errors.New` 是 CLAUDE.md 的规则例外（跨包 `errors.Is` 语义要求），注释已解释，但未登记为 approved exception（F3）。

### 3. 测试/回归

`nonce_test.go` 覆盖：first-use、replay、expired-reuse、concurrent-distinct、concurrent-same-nonce-exactly-one、lazy-prune、zero/negative maxAge、Kind values。`servicetoken_test.go` 覆盖：replay accepted/rejected、unique tokens、nil store。`controlplane_guard_test.go` 覆盖：dev/real mode miss、demo key current/previous、replay、TTL、real-mode Kind assert。`auth_integration_test.go` 覆盖：端到端 replay 401 全栈。多 pod 场景按计划 skip（Redis store 独立 PR），符合 plan 记录。集成测试子测试共享 nonceStore 无并发问题（当前无 `t.Parallel()`），但缺乏防止未来误加的注释（F6）。

### 4. 运维/部署

`GOCELL_HTTP_INTERNAL_ADDR` 文档默认值 `:9090` 与代码实现 `127.0.0.1:9090` 不一致（F1）——运维人员可能误以为 dev 部署默认绑定全网卡，属于安全相关文档错误。guard 安装成功后无 `slog.Info` 确认 nonce store kind（F4），使得多 pod 运维无法从启动日志判断是否使用了 in-memory（单 pod only）还是 distributed 实现。新增 `ErrControlplaneServiceSecretMissing` / `ErrControlplaneNonceStoreMissing` 映射到 `codeToStatus` 已完成，ops grep 可达。

### 5. 可维护性/DX

`WithServiceTokenNonceStore` → 从 `WithNonceStore` 改名后，godoc 说明清晰，nil 入参语义有注释。`NoopNonceStore` null-object 模式消除了之前整个 nil-check 分支，代码更干净。`writeServiceTokenError` 两分支的优先关系依赖有注释 `// errors.Is traverses the full chain, so ErrNonceReused in the Cause matches.`，但未说明"此检查必须先于 Cause 检查"的原因（F2）。测试名 `WithoutNonceStore` 在 PR 后语义漂移（F5）。

### 6. 产品/用户体验

replay 响应 `"service token replay detected"` + `ERR_AUTH_UNAUTHORIZED` 对消费方清晰可辨，便于调试。dev 模式默认安装 InMemoryStore 而非 Noop，dev 路径下重放也会得到 401——这对开发者是正确且无意外的行为，与 test helpers `freshTestServiceSecret` 配合自然。
