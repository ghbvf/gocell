- 工具：golangci-lint v2，logcheck.so（sigs.k8s.io/logtools），kube-api-linter.so（sigs.k8s.io/kube-api-linter），sorted.so（k8s.io/kubernetes/hack/tools/golangci-lint/sorted）

- 行为：先 validate config；分离 stderr（前缀 ERROR:）；TTY 彩色输出；patch plugin 路径；缓存加速
- exit code：0 = 通过；非 0 = violations，区分 mandatory fix（默认 config）vs optional（hints config）

**hack/verify-import-boss.sh**
- 工具：`go run ./cmd/import-boss`，依赖 `jq`
- 规则：读取目录下 `.import-restrictions` 文件定义的 import 白/黑名单
- 包发现：`go work edit -json | jq -r '.Use[].DiskPath + "/..."'`
- 环境：`GOPROXY=off`（禁止拉取外部模块）
- exit code：`set -o errexit`，任何命令失败立即退出

**hack/verify-featuregates.sh**
- 工具：`go run test/compatibility_lifecycle/main.go feature-gates verify`，`genfeaturegates`
- 规则：(1) feature gate 文档与代码定义一致；(2) lockedToDefault:true 的 feature 在删除前必须经历至少 3 个版本
- exit code：校验失败输出提示并以 1 退出；trap 清理临时文件

**hack/verify-test-featuregates.sh**
- 工具：`git grep`
- 规则：`*_test.go` 文件中禁止直接引用 `MutableFeatureGate`，必须通过 `featuregatetesting.SetFeatureGateDuringTest(t, utilfeature.DefaultFeatureGate, features.<Name>, <value>)`
- exit code：找到违规则以 1 退出

### 2.1 TestForbiddenImports — 完整禁令列表

每条格式：`禁 import X → 强制走 Y，理由`

1. `golang.org/x/net/context` → `context`，标准库已内置，使用 x/net/context 是旧习惯
2. `log` → `util/log`，使用项目统一结构化日志库
3. `github.com/golang/protobuf/proto` → `github.com/gogo/protobuf/proto`，项目采用 gogo protobuf
4. `github.com/satori/go.uuid` → `util/uuid`，使用项目 UUID 工具包
5. `golang.org/x/sync/singleflight` → `github.com/cockroachdb/cockroach/pkg/util/syncutil/singleflight`，使用项目封装版本
6. `syscall` → `sysutil`，使用项目封装的系统调用工具
7. `errors` → `github.com/cockroachdb/errors`，使用项目错误库（含 stack trace 等）
8. `oserror` → `github.com/cockroachdb/errors/oserror`，使用项目版本
9. `go.uber.org/atomic` → `sync/atomic`，标准库已够用
10. `github.com/pkg/errors` → `github.com/cockroachdb/errors`，统一错误处理
11. `github.com/cockroachdb/errors/*`（子包）→ `github.com/cockroachdb/errors`（顶层包），禁止直接引用子包
12. `util/log`（在 roachtest 中）→ `roachprod/logger`，roachtest 有自己的 logger
13. `util/ctxgroup`（在 roachtest 中）→ `roachtestutil/task`，使用 roachtest 专用 task 包
14. `golang.org/x/sync/errgroup`（在 roachtest 中）→ `roachtestutil/task`，同上

### 2.2 TestForbiddenAPI — 完整禁令列表

每条格式：`禁 API X，强制走 Y，理由`
