## §二 · v1.1 候选清单（3 个 Batch / ~10-13 周）

### Batch 1 — Codegen 全栈 + archtest 升级（**4 PR / ~5-7 周**，最高 ROI）

**主题**：把 GoCell 重框架感受度从 ~6× 降到 ~5×（hello world 7 文件 → 2-3 文件 / 220 行 → ~50 行 / 11+ 概念 → 5-6 概念）；archtest 从 string scan 升级为真类型分析。

| # | PR 名 | 关闭项 | 总分 | 工时 | 依赖 |
|---|---|---|---:|---:|---|
| B1-1 | **PR-V11-CODEGEN-CELL-GEN** | E3：cell.go marker → cell_gen.go（含 Cell skeleton + `Init(ctx, reg)` 方法 + 所有 `reg.X()` 调用 + 编译期 interface 断言）+ slice_gen.go（handler 函数桩 + service interface 骨架 + 路由声明）+ verify-codegen gate（`git worktree add` 隔离沙箱模式，K8s 范式） | 15 | 2-3 周 | 027 #31 (E2 Registry API) 必须先合 |
| B1-2 | **PR-V11-CODEGEN-MARKER-MIGRATE** | E4：cell.yaml + slice.yaml 真相源迁移到 Go 代码（marker 主 + struct embed 辅）；`gocell generate cell` 用 `go/ast` 解析 marker + struct embedded type，反向生成 yaml；CI verify 守护；contract.yaml + assembly.yaml + journey.yaml + actors.yaml **保留为真相源**（不参与 codegen） | 15 | 1-1.5 周 | B1-1（共用 codegen 引擎） |
| B1-3 | **PR-V11-CONTRACT-DTO-CODEGEN** | E10：contract.yaml 加 OpenAPI-style 完整 schema → `gocell generate contracts` 自动产出 `generated/contracts/<id>/{types,iface,errors}.go`；业务 service.go 实现 generated interface 即可，不再手写 DTO/iface/errcode 常量 | 15 | 1.5-2 周 | B1-1（共用 codegen 引擎） |
| B1-4 | **PR-V11-DEPGRAPH-TOOLS** | PR-A38：新模块 `tools/depgraph/`（基于 `golang.org/x/tools/go/packages`）+ Layer/CellID 推断 + JSON/DOT 输出；解锁 archtest 真类型分析（取代文件级 string scan）；同步 LAYER-GO-IMPORT-01 升级到传递闭包级校验 | 11 | 1.5-2 周 | 与 B1-1/2/3 文件域不冲突，全程并行 |

**Batch 1 总效果**（参照 `engineering-baseline/202604300600` §3 减重数学）：
- hello world 文件数: 7 → **2-3**（仅 contract.yaml + service.go + 可选 cell.go marker）
- 手写行数: ~220 → **~50**
- 必填 yaml 字段: 13 → **0**
- 入场时间: 4-8h → **1-2h**（B2 配合 E9 后 → 5-10min）
- 比值 vs net/http: ~18× → **~6×** → B2 后 **~5×**
