**依赖**: 无前置 PR（可并行于 PR-ARCH-LAYER）

| # | 任务 | 优先度 | 工时 |
|---|------|--------|------|
| L2 | **ROUTE-POLICY-REGISTRY-01** — 实现路由策略注册表，要求 `RouteDecl` 携带三选一安全语义（`Policy` / `Public=true` / `Delegated=true`）；`runtime/http/router/policy_registry.go` 新增 `RecordRoute` + `FinalizePolicyRegistry`；bootstrap 启动期调用，发现裸路由 fail-fast | P1 | 5h |

> **设计参考**（见审查报告 §6）：
> - Kubernetes: 全局过滤链 + 白名单显式声明
> - chi: Walk() 路由树遍历实现启动期审计
> - Casbin: 策略判定引擎（不能单独防裸路由）
>
> **推荐落地形式**:
> ```go
> // RouteDecl 新增安全语义字段
> type RouteDecl struct {
>     Method  string
>     Path    string
>     Handler http.Handler
>     // 三选一，缺失则 FinalizePolicyRegistry() 返回 error
>     Policy    PolicyFunc   // 业务端点必须提供
>     Public    bool         // /healthz /readyz /metrics 等
>     Delegated bool         // /internal/v1/* 走 ServiceToken
> }
> ```

## 风险备注

1. **S-nonce 需要 Redis**: `WithNonceStore` 的真实实现需要 `adapters/redis/` NonceStore；若 Redis 接线尚未就绪，可先实现 in-memory 窗口 store（单 Pod 可接受），PG 上线后换 Redis 实现
2. **S23 bootstrap 拆分工时可能超估**: sso-bff example 的 bootstrap 分叉较深，需要预留额外时间；若超出则 PR 拆分为 fix-crash（L7-ex，30min）和 bootstrap-unify（S23，独立 PR）
3. **F10 Journey Harness 依赖 PG+RMQ 可用**: 若 testcontainers 在 CI 中不稳定，可降级为 in-memory adapter 先实现 3 条核心 Journey，其余延后
4. **L2 PolicyRegistry 对存量路由的影响**: 实施 L2 时需要确保所有现有路由都已正确标记 Public/Delegated/Policy，否则 fail-fast 会阻断启动；建议先在 warn 模式运行一个 PR，确认覆盖率后再切 error 模式
