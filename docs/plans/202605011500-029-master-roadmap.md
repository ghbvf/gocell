> **优先级共识**：Lane K（关键路径）+ Lane G（CI/lint/治理）先行；Lane A/B/C/D 在 K Phase 2-3 期间被动暂停，待 K 装备类 Phase 4 之后再启。理由：K 决定 API 形状（后续所有 lane 必须基于其 API 写代码），G 决定 lint baseline（先把口子收紧，后续 PR review 才有静态保护、不靠人肉）。

- 先把 lint baseline 拉到 0 issues，K #01 之后所有 PR（含 K 自身）就能借力静态守卫；否则 K #04-#06 codegen 产物可能引入 ~百条 lint finding，到 Phase 3 末端再修代价 5-10 倍
