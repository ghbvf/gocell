## 设计原则

1. **延续 025 plan 的文件亲缘 + 语义内聚**：同 backlog 主题 + 同目录的多条小修打包成一个 PR
2. **新编号单调**：PR-A39 起严格递增，避免与 025 的 PR-A21/A22/A23/A24/A33 长期占位冲突
3. **触发条件项单独 Wave**：Wave 9（多 pod / 发布前）+ Wave 10（command 生产治理）不进开发主线，触发后再启动
4. **Won't-do 显式登记**：5 条「评估后不做」必须写入 §六，附决策依据；下一份 plan 不再回看
5. **共享子项合并**：跨 backlog 项的同一文件改动（如 PR250-F2 + PR247-N3 都需要 `user_events.go` 抽 typed struct）合并到一个 PR，避免冲突
