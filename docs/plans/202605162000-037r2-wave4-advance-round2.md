## 核心论点修正（R1 → R2）

037 R1 的筛选轴是「**trigger 距离 = 1**（下一个就触发）→ 主动做 ROI > 等触发」。R2 纠正并外延这一轴：

> 真正的判据不是「trigger 是否触发」，而是「**scope 是否已知**」。把「gate 未满足」当「不可行 blocker」是 lazy deferral 的包装。Wave 4 剩余项据此二分：
>
> - **A 类 — 真 scope-blocker**：触发信号本身定义正确 scope（误报案例 / 事故形态）。信号未现 = 连规则要拦什么都不确定，先做 = 凭猜 = PR #445 反模式。**保持休眠**。
> - **B 类 — 优先级/决策 gate**：scope 已在 backlog 写死，「未做」是优先级/ROI 选择或一次内部决策，**不是外部阻塞**。"方案未定" 对决策 gate 而言是任务第一步，非 blocker。**全部可主动推进**，挡它们的唯一诚实理由是机会成本 + 单 PR=单维度的 scope 纪律。

**适用范围**：仅 036 §4 Wave 4 B 类。A 类 + 大体量 Template-Wave4-3PR 项不在本波。

### R2-P1：PR-TS1-FU-VALIDATIONRESULT-EMITTER-SEALED-MARKER-01 ✅ shipped 2026-05-16

> **Ship 报告（含 reviewer F-1 闭环）**：`kernel/governance/emitter_invariant.go` 声明 BFS owner invariant + 3 行 compile-time witnesses；`signatureMatchesValidationResultEmitter` predicate body 三处全 `types.Identical`（含 owner gate `types.Identical(recvNamed, locatorType)`），**零字符串锚点 + 结构性免疫 method promotion**；fixture loader fail-fast on missing scope identifier；新增 2 个 RED fixture（embedded_locator outer receiver + unrelated receiver）覆盖完整威胁面。**Hard 防线四层**：type-identity owner gate + compile-time witnesses + predicate body type-system + loader fail-fast。OSS 对标超过 staticcheck/govet/gopls 两档。
>
> **设计演化**：首版（PR #521 commit baba2a6f2）用 sealed marker iface (`validationResultEmitter` + `*locator.isValidationResultEmitter()`) + `types.Implements` 做 owner gate。Reviewer F-1 (Cx1) 发现 Go embedding 自动 promote marker method 到 *Validator / *DependencyChecker（二者均 `struct { locator }` 值嵌入），导致 `types.Implements(*Validator, iface)=true`，owner gate 失效。改为 `types.Identical(recvNamed, locatorType)`——named type 身份不参与 promotion，结构性免疫此攻击面。Sealed marker 是 cross-package unimplementability 范本（kernel/outbox / kernel/persistence），不适合 same-package owner allowlist。

- **现状**：PR-TS1 把 BFS emitter 升至 signature-based 三谓词（return ValidationResult + arg0 string + receiver/result 同包），AI-rebust Medium-偏-Hard；owner 约束「同包」比原 `*locator` 单一允许集略宽 → 同包新加非-`*locator` 的 `(string,...) ValidationResult` 方法会被误纳（真 Soft false-positive 面）。
- **修复**：`kernel/governance` 加 unexported sealed marker `validationResultEmitter interface { isValidationResultEmitter() }`，`*locator` 实现；BFS predicate 改 `types.Implements(recvNamed, emitterIface)`。
- **维度**：单维 framework（predicate 改写 + 既有 `rule_inventory_test.go` conformance 同一 surface，无独立 enforcement PR）。
- **AI-rebust**：Medium-偏-Hard → **真 Hard**（违反不可表达——非 emitter 类型编译期不实现 marker）。
- **文件**：`kernel/governance/locator.go` + `kernel/governance/rule_inventory_test.go`。
- **前置**：无。**关闭 backlog** `cap-02-metadata-governance.md` PR-TS1-FU-… 行（🟠 → ✅）。

- **P5.1 实施**（单 PR、单维 enforcement，~3-4h）：重写 `tools/archtest/cellgen_errcode_funnel_test.go` 到 types.Info-driven 白名单 form uniqueness——扫 cellgen 包内 CallExpr，按 *types.Info 解析 callee（覆盖 SelectorExpr/Ident/dot import 所有形态），若 callee 返回类型含 `error`，则 callee 必须是 *types.Func 且 `.Pkg().Path() == "github.com/.../pkg/errcode"`；解析到 *types.Var（function-valued re-export）显式 fail。**不依赖 callee name pattern**（详见 ADR §D5 论证：name pattern 漏 dot import / 裸 Ident / re-export，且漏白名单成员如 Assertion）。
