# Cellular PRD：Rust 优先的 Cell-Native 框架

> 状态：Draft / Discussion
> 作者：Architecture WG
> 日期：2026-05-09
> 范围：用 Rust 原生设计哲学重新打造 cell-native 框架。GoCell 是灵感来源，不是移植目标
> 项目代号：**Cellular**（占位，可改）
> 非范围：与 Go 版 GoCell 的 wire/yaml/二进制兼容；Go ↔ Rust 互调

---

## 0. 摘要（TL;DR）

Cellular 是一个**用 Rust 类型系统从零设计的 cell-native 框架**，受 GoCell 启发但不背任何兼容包袱。核心命题：

> **Rust type system + GAT + const generic + RAII + proc macro 能让 cell-native 治理 90%+ 在编译期表达，dylint/lint 规则 ≤ 8 条。**

设计哲学优先级（高→低）：

1. **类型系统拦住** > proc macro 派生 > dylint 兜底 > 文档约定
2. **零成本抽象**（generic monomorphization）> trait object（仅在跨边界注入时使用）
3. **RAII 资源管理** > 显式 init/start/stop 回调
4. **enum variant 区分 wire vs server 路径** > 三层 redaction 字段
5. **derive macro 单一来源** > 外部 yaml schema
6. **cargo workspace + crate visibility** > archtest 守卫

按 L0→L4 一致性等级分 6 个垂直切片（M0–M5），每个切片是端到端可运行 demo。

## 1. 背景与动机

### 1.1 为何不直接移植 GoCell

GoCell 的 23 kernel 子包 + 74 archtest + cell.yaml/slice.yaml/contract.yaml 三层 schema，**70% 是为了弥补 Go 语言能力的 gap**：

| GoCell 设计 | Go 的 gap | Rust 的对应能力 |
|---|---|---|
| `cell.yaml` 声明式元数据 | 没有 macro | proc macro derive |
| `archtest MESSAGE-CONST-LITERAL-01` | 没有 `&'static str` 参数 | 函数签名直接表达 |
| `errcode` 三层 Message/Details/Internal | `error` 是 `interface{}` | enum variant + sealed trait |
| `wire-format-out-of-kernel` ADR | 反射序列化无法限制字段 | `impl Serialize` 手写 + visibility |
| `OUTBOX-LEASE-ID-CAS-01` archtest | 没有 visibility 控制单字段 | private field + RAII guard |
| `lifecycle.ContextCloser` | 没有 RAII | `impl Drop` |
| `kernel/ctxkeys` 类型化 key | `context.Context` 是 `interface{}` | tracing span fields + newtype |
| 工作区分层 archtest | 包可见性弱 | cargo crate visibility |

**直接移植会把 Go 语言的工程包袱搬进 Rust，浪费 Rust 类型系统的杠杆。** 所以 Cellular 是重新设计，不是移植。

### 1.2 保留什么、抛弃什么

| 维度 | 保留（GoCell 的真领域 IP） | 抛弃（Go 工程包袱） |
|---|---|---|
| 一致性等级 L0–L4 | ✅ 概念保留，用 const generic / marker trait | ❌ 不再用 yaml 字段 |
| Cell + Slice 边界 | ✅ 概念保留，用 cargo crate | ❌ 不再用 cell.yaml/slice.yaml |
| Contract 跨边界 | ✅ 概念保留，用 trait + 关联类型 | ❌ 不再用 contract.yaml + codegen |
| Outbox + 幂等 | ✅ 模式保留，用 Stream + Lease RAII | ❌ 不再有 Claim/Commit/Release 显式 API |
| 错误三层（PII safe） | ✅ 语义保留，用 enum variant + 序列化对偶 | ❌ 不再有 Message/Details/Internal 三字段 |
| Cell label / metrics | ✅ 保留，用 tracing span field 单源 | ❌ 不再有 ctxkeys 单独包 |
| 治理闭环 | ✅ 保留概念，用类型 + 8 条 dylint 兜底 | ❌ 抛弃 74 个 archtest 中的 ~60 个 |
| Journey 验收规格 | ✅ 保留，用 cargo test + integration | ❌ 不再有 J-*.yaml 独立 schema |

### 1.3 非目标

- ❌ wire 兼容 GoCell（Cellular 的 event/http schema 是新的）
- ❌ 复用 GoCell 的 cell.yaml / contract.yaml / slice.yaml
- ❌ 复用 GoCell 的 codegen 工具链
- ❌ 双向 Go/Rust 互调
- ❌ WASM / no_std
- ❌ 性能基准 vs Go GoCell（验证设计可行性）

## 2. Rust 优先设计哲学（贯穿全 PRD）

### 2.1 类型系统作为第一道防线

每条治理约束首先问：**能否让违反在编译期不可表达？** 能 → 用类型；不能 → 用 proc macro；都不能 → 用 dylint。

**例：PII 不泄漏到 wire**

GoCell：archtest `MESSAGE-CONST-LITERAL-01` + ADR + code review。
Cellular：

```rust
#[derive(thiserror::Error, Debug)]
pub enum NoteError {
    #[error("note not found")]                           // 客户端可见
    NotFound { id: NoteId },                              // id 是 PII 安全（UUID）
    
    #[error("validation failed")]
    Invalid(#[source] ValidationError),                   // 错误链可见
    
    #[error("internal storage error")]
    Internal(#[source] anyhow::Error),                    // 仅 server 日志，wire 屏蔽
}

impl IntoResponse for NoteError {
    fn into_response(self) -> Response {
        match self {
            Self::NotFound { id } => json_error(404, "NOT_FOUND", json!({"id": id})),
            Self::Invalid(e)      => json_error(400, "INVALID",   json!({"reason": e.to_string()})),
            Self::Internal(_)     => json_error(500, "INTERNAL",  json!({})),  // 不下发 source
        }
    }
}
```

`Internal(source)` 的 `source` 字段类型是 `anyhow::Error`，**没有 `Serialize` 实现**——故意让序列化在编译期失败。`Display` impl 仅供 `tracing::error!` 记录，不进 wire。**0 行 archtest**。

### 2.2 RAII 替代显式生命周期 API

**例：Outbox lease 不可泄漏**

GoCell：archtest `OUTBOX-LEASE-ID-CAS-01` + archtest `HANDLER-RECEIPT-WRITE-01` + `kernel/idempotency.Claimer` 包。
Cellular：

```rust
pub struct Lease<'a, E: Event> {
    entry_id: EntryId,
    payload: E::Payload,
    inner: LeaseInner<'a>,           // 私有，含 lease_id (UUID)
}

impl<'a, E: Event> Lease<'a, E> {
    pub fn payload(&self) -> &E::Payload { &self.payload }
    pub async fn ack(self) -> Result<()> { /* CAS lease_id, mark_published */ }
    pub async fn nack(self, requeue: bool) -> Result<()> { /* CAS lease_id */ }
}

impl<'a, E: Event> Drop for Lease<'a, E> {
    fn drop(&mut self) {
        // 未显式 ack/nack → 自动 requeue + 触发 metric
    }
}
```

Handler 拿到的是 `Lease<E>`，**LeaseId 不在公共 API**，无法泄漏；忘记 ack 由 Drop 自动 requeue（fail-closed）。**0 行 archtest，0 行 dylint。**

### 2.3 const generic / marker trait 表达一致性等级

GoCell：`consistency: L2` yaml 字段 + 文档约定 "L1 cell 不能 emit event"。
Cellular：

```rust
pub trait ConsistencyLevel: sealed::Sealed { const KIND: &'static str; }
pub struct L0; impl ConsistencyLevel for L0 { const KIND: &'static str = "LocalOnly"; }
pub struct L1; impl ConsistencyLevel for L1 { const KIND: &'static str = "LocalTx"; }
pub struct L2; impl ConsistencyLevel for L2 { const KIND: &'static str = "OutboxFact"; }
pub struct L3; impl ConsistencyLevel for L3 { const KIND: &'static str = "WorkflowEventual"; }
pub struct L4; impl ConsistencyLevel for L4 { const KIND: &'static str = "DeviceLatent"; }

pub trait AtLeastL2 {} impl AtLeastL2 for L2 {} impl AtLeastL2 for L3 {} impl AtLeastL2 for L4 {}

// emit_event 仅在 L2+ 可调用
impl<C: AtLeastL2> CellCtx<C> {
    pub async fn emit<E: Event>(&self, event: E) -> Result<()> { ... }
}
```

L1 cell 调 `emit_event` → 编译期 `the trait AtLeastL2 is not implemented for L1`。**0 行 archtest。**

### 2.4 derive macro 是单一来源

GoCell：cell.yaml + slice.yaml + contract.yaml + verify.contract + cell_gen.go，**5 处声明同步**，靠 ADV-06 / VERIFY-01 archtest 守。
Cellular：

```rust
#[cell(id = "noteshelf", consistency = L1)]
#[contracts(
    serves(NoteCreateV1),
    serves(NoteQueryV1),
    emits(NoteCreatedV1),                  // L2 才可写，编译期检查
)]
pub struct Noteshelf {
    db: PgPool,
}
```

derive macro 唯一展开点：注册、subscription declaration、metrics label、verify hook 全部从这里派生。**漂移在物理上不可能**——只有一处可改。

### 2.5 cargo workspace 是 cell 边界

GoCell：archtest `CELL-INIT-NO-RUNTIME-IMPORT` 守 import。
Cellular：

```toml
# crates/cells/searchcore/Cargo.toml
[dependencies]
cellular-kernel = { path = "../../kernel" }
cellular-contracts = { path = "../../contracts" }
# noteshelf 不在 dependencies → cargo build 阻断 use noteshelf::...
```

跨 cell 直接 import 业务类型 → cargo 报 `crate not found`。**0 行 archtest。**

### 2.6 异步抽象用 Stream / Sink / Future，不发明新接口

GoCell：自创 `Subscriber`、`Publisher`、`Receipt`、`Settlement` 等 5+ 接口。
Cellular：

```rust
pub trait OutboxStore: Send + Sync + 'static {
    fn enqueue<E: Event>(&self, event: E) -> impl Future<Output = Result<EntryId>> + Send;
    fn drain<'a, E: Event>(&'a self) -> impl Stream<Item = Lease<'a, E>> + Send;
}
```

`Stream` 是 Rust 标准异步接口（futures crate / std stream），所有 Rust 开发者一眼看懂。

### M0：Foundation（L0 + L1）

#### 目标

把 Cellular 的**核心心智模型**全栈跑通：cell trait + contract trait + derive macro + axum + sqlx + RAII 生命周期 + enum-variant 错误模型。

#### Demo：`noteshelf`

单 cell `noteshelf`，3 个 slice：

| Slice | 一致性 | 类型签名（示意） |
|---|---|---|
| `noterender` | `L0` | `fn render(md: &str) -> Html` 纯函数 |
| `notecreate` | `L1` | `async fn create(ctx: CellCtx<L1>, req: NoteCreateReq) -> Result<NoteCreateRes, NoteError>` |
| `notequery` | `L1` | `async fn query(ctx: CellCtx<L1>, q: NoteQuery) -> Result<NoteList, NoteError>` |

#### 关键技术决策

**1. Cell trait（核心 API）**

```rust
pub trait Cell: Send + Sync + 'static {
    const ID: &'static str;
    type Consistency: ConsistencyLevel;
    type Config: DeserializeOwned + Send + Sync;
    type Error: ContractError;
    
    async fn build(cfg: Self::Config, deps: BuildCtx<'_>) -> Result<Self, Self::Error>
    where Self: Sized;
    
    async fn run(self: Arc<Self>, ctx: RunCtx) -> Result<(), Self::Error>;
}
```

`build` 替代 GoCell 的 `Init`；`run` 替代 `Start`；`stop` **不存在**——shutdown 由 `RunCtx` 内的 `CancellationToken` 协作完成，资源清理由 `Drop` 接管（RAII）。

**2. Contract trait + derive**

```rust
pub trait HttpContract {
    const ID: &'static str;
    const METHOD: http::Method;
    const PATH: &'static str;
    type Request: DeserializeOwned + Send;
    type Response: Serialize + Send;
    type Error: ContractError;
}

#[http_contract(method = POST, path = "/api/v1/notes")]
pub struct NoteCreateV1;

impl HttpContract for NoteCreateV1 {
    type Request = NoteCreateReq;
    type Response = NoteCreateRes;
    type Error = NoteError;
}
```

**3. CellCtx 用 const generic 锁一致性等级**

```rust
pub struct CellCtx<'a, C: ConsistencyLevel> {
    db: &'a PgPool,
    span: &'a tracing::Span,
    _consistency: PhantomData<C>,
}

// 仅 L1+ 提供事务
impl<'a, C: AtLeastL1> CellCtx<'a, C> {
    pub async fn tx<F, T>(&self, f: F) -> Result<T> where F: ... { ... }
}
```

**4. 错误模型：每 cell 一个 enum**

每个 cell 定义自己的 `Error` enum，wire 序列化由 `IntoResponse` 控制（不暴露 `Internal` source）。**没有全局 errcode 包**——因为 Rust 的 enum 比共享错误类型更类型安全。

**5. 数据库**：sqlx 0.8 + `query!` 宏 + offline mode（CI 必须 schema cache）。

**6. 配置**：`figment`（toml + env），cell 配置由 `Cell::Config` 关联类型类型化。

#### 引入 derive macro（M0 必须）

```rust
#[derive(Cell)]
#[cell(id = "noteshelf", consistency = L1)]
pub struct Noteshelf { db: PgPool }
```

这一步是**与 GoCell 移植版的核心差异**——M0 起就用 macro，不走 yaml 过渡。

#### 不引入

- ❌ outbox / 事件
- ❌ 跨 cell 通信
- ❌ otel / prometheus（仅 `tracing` stdout）
- ❌ vault（hard-coded JWT secret）
- ❌ ratelimit / circuitbreaker

#### 验收标准

```bash
cargo run -p noteshelf
curl -X POST http://localhost:8080/api/v1/notes -d '{"title":"x","body":"# y"}'    # 201
curl http://localhost:8080/api/v1/notes/{id}                                         # 200
curl http://localhost:8080/healthz; curl http://localhost:8080/readyz                 # 200/200
```

**Rust 优先验收（关键）**：

1. 故意让 L0 slice 调 `ctx.tx(...)` → **编译错误**：`AtLeastL1 not implemented for L0`
2. 故意 `serde_json::to_string(&NoteError::Internal(...))` → **编译错误**：`anyhow::Error` 无 `Serialize`
3. 故意忘记调 `Cell::build` 注册 → **编译错误**：`register_cells!` 宏未引用 → 链接期 `__CELL_NOTESHELF` 缺失
4. `cargo run -p noteshelf` 输出 tracing 含 `cell="noteshelf"` 字段
5. 单元测试 + integration（testcontainers + PG）覆盖 ≥ 80%

### M1：Events（L2 OutboxFact）

#### 目标

证明 **Lease<E> RAII guard + Stream<Lease> 抽象** 能干净表达事务性事件 + 幂等。

#### Demo

`noteshelf::notecreate` 升级到 L2：本地事务内写 `notes` 表 + 写 outbox，relay 异步发 `note.created.v1` 到 RabbitMQ。

#### 关键技术决策

**1. Event trait + derive**

```rust
pub trait Event: Serialize + DeserializeOwned + Send + Sync + 'static {
    const ID: &'static str;        // "note.created.v1"
    const TOPIC: &'static str;     // routing key
}

#[derive(Event, Serialize, Deserialize)]
#[event(id = "note.created.v1", topic = "note.created.v1")]
pub struct NoteCreatedV1 { pub id: NoteId, pub title: String }
```

**2. CellCtx<L2> 才能 emit**

```rust
impl<'a, C: AtLeastL2> CellCtx<'a, C> {
    pub async fn emit<E: Event>(&self, event: E) -> Result<EntryId, EmitError> { ... }
}
```

L1 cell 调 emit → 编译错误。

**3. OutboxStore trait（Stream-based）**

```rust
pub trait OutboxStore: Send + Sync + 'static {
    fn enqueue<E: Event>(&self, tx: &mut Tx<'_>, event: &E) -> impl Future<Output = Result<EntryId>>;
    fn drain<'a>(&'a self) -> impl Stream<Item = Result<RawLease<'a>>> + Send + 'a;
}

pub struct RawLease<'a> {
    entry_id: EntryId,
    contract_id: &'static str,
    payload: Bytes,
    inner: LeaseInner<'a>,                     // private，含 lease_id
}

impl<'a> RawLease<'a> {
    pub fn typed<E: Event>(self) -> Result<Lease<'a, E>, DecodeError> { ... }
    pub async fn ack(self) -> Result<()> { ... }
    pub async fn nack(self, requeue: bool) -> Result<()> { ... }
}

impl<'a> Drop for RawLease<'a> {
    fn drop(&mut self) {
        // 未 ack/nack → 异步任务 requeue + 计数 metric
    }
}
```

**4. Lease 是 RAII**：忘记 ack → Drop 触发 requeue（fail-closed）。LeaseId 不在公共 API。

**5. Disposition 用 self-consuming 方法替代 enum**：

GoCell 用 `enum Disposition { Ack, Requeue, Reject }` + `HandleResult { disposition, err }`。Rust 优先：直接调 `lease.ack() / lease.nack(requeue)`，不需要枚举值——方法即语义，不会有"零值是 invalid Disposition"问题。

**6. Permanent error 的处理**：handler 返回 `Result<(), HandlerError>`，HandlerError 区分 `Permanent(e)` / `Transient(e)` / `Reject(reason)`，consumer base 据此决定 `lease.ack()` / `lease.nack(true)` / `lease.nack(false)`。

**7. PostgreSQL outbox + lease fencing**：

```sql
UPDATE outbox_entries
SET status = 'published', lease_id = NULL
WHERE entry_id = $1 AND lease_id = $2;       -- CAS 守
```

#### 不引入

- ❌ 跨 cell consumer（M2）
- ❌ 多 broker（仅 RabbitMQ via lapin）
- ❌ 多 outbox 后端（仅 PG）

#### 验收标准

```bash
docker compose up -d postgres rabbitmq
cargo run -p noteshelf
curl -X POST .../notes -d '...'                            # 201
psql -c "select status, lease_id from outbox_entries"     # pending→published
rabbitmqadmin get queue=note.created.v1
```

**Rust 优先验收**：

1. 故意 `let _ = lease;` 不调 ack → drop 触发 requeue，metric `outbox_lease_dropped_total` +1
2. 故意 handler 中 `let id = lease.inner.lease_id;` → 编译错误：`field is private`
3. handler 在 `.ack()` 后再用 lease → 编译错误：`use of moved value`
4. L1 cell 调 `ctx.emit(...)` → 编译错误：`AtLeastL2 not implemented`
5. 旧 worker 持过期 lease 调 ack → SQL CAS 返回 0 行 → 自动转 nack
6. handler 返回 `Err(HandlerError::Permanent(_))` → DLX 队列有消息

### M2：CrossCell（L3 WorkflowEventual）

#### 目标

证明 **跨 cell 通信 = `cargo workspace member` + `Subscriber<E>` trait**，无需任何 yaml/codegen。

#### Demo

新 cell `searchcore`，订阅 `note.created.v1`，写本地投影表 + 暴露 `GET /api/v1/search?q=`。

#### 关键技术决策

**1. Subscriber trait**

```rust
pub trait Subscriber<E: Event>: Cell {
    async fn handle(&self, ctx: HandlerCtx<'_, Self::Consistency>, event: E)
        -> Result<(), HandlerError>;
}

#[async_trait]
impl Subscriber<NoteCreatedV1> for Searchcore {
    async fn handle(&self, ctx: ..., event: NoteCreatedV1) -> Result<(), HandlerError> {
        ctx.tx(|tx| async move { /* upsert projection */ }).await
    }
}
```

`HandlerCtx::tx` 自动绑定本地事务，`Subscriber` trait 内部由 framework drain Stream<Lease<E>> + 转换为 typed event。

**2. cell 边界由 Cargo.toml 锁定**

```toml
# crates/cells/searchcore/Cargo.toml
[dependencies]
cellular-kernel.workspace = true
cellular-contracts.workspace = true
# noteshelf 不在依赖列表
```

`searchcore` 写 `use noteshelf::Note;` → cargo 报 `unresolved import`。

**3. Contract crate 是唯一共享单元**

```
crates/contracts/    # cellular-contracts，含所有 Event / HttpContract trait 定义
├── note/
│   ├── created_v1.rs
│   └── ...
```

cell 之间不可直接引用对方业务类型，**唯一共享通道是 contract types**。

**4. SubscriptionRegistry derive 派生**

```rust
#[cell(id = "searchcore", consistency = L3)]
#[contracts(subscribes(NoteCreatedV1))]
pub struct Searchcore { ... }
```

derive 宏展开：

- 生成 `__CELLULAR_SUBSCRIPTION_SEARCHCORE_NOTECREATEDV1: SubscriptionDecl`（链接期发现）
- 编译期 require trait `Subscriber<NoteCreatedV1>` for `Searchcore`，缺失 → 编译错误
- `cellular-cli graph` 静态分析所有 `__CELLULAR_SUBSCRIPTION_*` 符号，输出 cell 依赖图

**5. handler 不接触 lease**

framework 内部 drain `Stream<Lease<E>>`，`lease.typed::<E>()` 后调 `subscriber.handle(event)`，根据 `Result` 自动 `lease.ack()` / `lease.nack(requeue)`。**handler 永远拿不到 Lease 类型**。

#### 不引入

- ❌ L4
- ❌ 多投影 store（PG only）
- ❌ 实时搜索（最终一致即可）

#### 验收标准

```bash
docker compose up -d
cargo run -p noteshelf &
cargo run -p searchcore &
curl -X POST .../notes -d '{"title":"rust","body":"..."}'
sleep 1
curl http://localhost:8081/api/v1/search?q=rust    # 返回 hit
```

**Rust 优先验收**：

1. 故意删 `impl Subscriber<NoteCreatedV1> for Searchcore` → **编译错误**：`#[contracts(subscribes(NoteCreatedV1))]` 要求该 impl
2. 故意在 searchcore/src/lib.rs 写 `use noteshelf::Note;` → **cargo build 失败**
3. `cellular graph` 输出 `noteshelf -[note.created.v1]→ searchcore`
4. 故意把 `NoteCreatedV1` 字段 `id` 类型从 `NoteId` 改成 `String` → noteshelf / searchcore 同时编译失败（contract 单源）

### M3：Commands（L4 DeviceLatent）

#### 目标

证明**长延迟单播命令 + ack 路由**用 Rust 标准 async + correlation token 干净表达。

#### Demo

新 cell `devicecore` + 独立 binary `device-sim`：

| Slice | 一致性 | 能力 |
|---|---|---|
| `commandissue` | L4 | POST `/devices/{id}/commands`，命令落 queue + outbox |
| `commandack` | L4 | PUT `/commands/{cid}/ack`，更新状态 |

`device-sim` 通过 WebSocket 接命令，1–10s 后随机 ack。

#### 关键技术决策

**1. Command vs Event 用不同 trait**

```rust
pub trait Command: Serialize + DeserializeOwned + Send + Sync + 'static {
    const ID: &'static str;
    type Reply: Serialize + DeserializeOwned + Send;
}

// 不同 trait → 不同 const generic 路径 → 编译期不可混淆
impl<C: AtLeastL4> CellCtx<'_, C> {
    pub async fn dispatch<Cmd: Command>(&self, target: DeviceId, cmd: Cmd)
        -> Result<CommandHandle<Cmd>, DispatchError> { ... }
}

pub struct CommandHandle<Cmd: Command> {
    correlation_id: CommandId,
    _phantom: PhantomData<Cmd>,
}

impl<Cmd: Command> CommandHandle<Cmd> {
    pub async fn await_reply(self, timeout: Duration) -> Result<Cmd::Reply, AwaitError> { ... }
}
```

**2. Reply 路由用 correlation_id (UUID)**：command_id 是 `CommandHandle` 内部字段，handler 持有以等待 ack；`device-sim` 回 ack 时携带 correlation_id；framework 用 `tokio::sync::oneshot` channel pool 路由回原 handler。

**3. Frozen Clock 测试 L4 长延迟**：

```rust
#[tokio::test(start_paused = true)]
async fn command_timeout_then_retry() {
    let ctx = test_ctx();
    let handle = ctx.dispatch(device_id, Reboot).await.unwrap();
    tokio::time::advance(Duration::from_secs(120)).await;
    let err = handle.await_reply(Duration::from_secs(60)).await.unwrap_err();
    assert!(matches!(err, AwaitError::Timeout));
}
```

`tokio::time::pause()` 是 Rust 异步测试事实标准，比 GoCell 自建 Clock 抽象简洁。

#### 不引入

- ❌ MQTT（仅 WebSocket）
- ❌ Device fleet 管理 / shadow state
- ❌ 多设备并发命令调度

#### 验收标准

```bash
cargo run -p devicecore &
cargo run -p device-sim -- --device-id=dev-01 &
curl -X POST .../devices/dev-01/commands -d '{"action":"reboot"}'
sleep 6
curl .../commands/{cid}    # status: acked
```

**Rust 优先验收**：

1. 故意把 `Reboot` 当 Event 给 `ctx.emit` → 编译错误（trait 不同）
2. L3 cell 调 `ctx.dispatch` → 编译错误：`AtLeastL4 not implemented for L3`
3. `tokio::time::advance` 测试覆盖超时 / 重试 / DLX 路径
4. correlation_id 重复 ack → second ack 返回 `Err(AlreadyAcked)`

### M4：Tooling（CLI + dylint 兜底 + scaffold）

#### 目标

提供完整工程化工具链，**收敛剩余无法用类型系统表达的约束到 ≤ 8 条 dylint**。

#### Demo

`cellular-cli` 安装后：

```bash
cellular new-cell mycell --consistency L1
cellular check                          # 跑全部 lint + 类型检查
cellular graph --format=dot > deps.dot   # cell 依赖图
cellular scaffold contract event/my/topic/v1 --payload=MyPayload
```

#### 关键技术决策

**1. proc macro 错误诊断（非协商）**

每条 derive 错误必须：

- 用 `proc-macro-error` 输出
- span 指向用户源码行而非宏内部
- `trybuild` UI test 覆盖每条诊断
- 错误信息含修复建议

**2. dylint 规则集（目标 ≤ 8 条）**

| # | 规则 | 替代 GoCell archtest | 为何不能 type 拦 |
|---|---|---|---|
| 1 | `cellular_redaction_sealed` | 跨 redaction / log / outbox last_error 三处单源 | redaction 在 trait object 边界，单 trait 难锁 |
| 2 | `cellular_cell_label_from_span` | metrics cell label 必须读 tracing span | metrics 中间件签名是 trait object |
| 3 | `cellular_no_direct_sqlx_in_cell` | cell 不能直接 import sqlx，必须走 `Repo` trait | 项目级架构约束，dylint 看 use 路径 |
| 4 | `cellular_panic_whitelisted` | bare `panic!` 仅在白名单文件 | clippy 不允许细化白名单 |
| 5 | `cellular_no_println_in_runtime` | runtime 代码不准 `println!` | clippy::print_stdout 没有粒度 |
| 6 | `cellular_test_time_clock` | 测试代码不准 `std::time::Instant::now()` | 业务代码可以，测试不行，需 ast 看上下文 |
| 7 | `cellular_no_unsafe_unless_doc` | unsafe 块必须有 SAFETY 注释 | clippy 类似规则不强制注释 |
| 8 | `cellular_async_no_block_on` | async 函数体内禁止 `block_on` | 类型上无法表达 |

**74 → 8，由类型系统消化 ~66 条。**

**3. CLI 命令**

| 命令 | 实现 |
|---|---|
| `new-cell` | tera 模板 + cargo workspace member 注册 |
| `check` | `cargo check` + `cargo clippy` + `cargo dylint` 串联 |
| `graph` | 解析所有 `__CELLULAR_*` 符号 + petgraph + DOT 输出 |
| `scaffold contract` | 生成 contract trait impl 模板（不是 yaml） |
| `verify` | 跑 cargo test + 自定义 invariant 检查 |

**4. workspace 模板**

```
my-app/
├── crates/
│   ├── kernel/         # cellular-kernel 引用
│   ├── contracts/      # 业务 contract 定义
│   └── cells/
│       ├── noteshelf/
│       └── searchcore/
├── examples/
│   └── docker-compose.yml
├── Cargo.toml          # workspace
└── cellular.toml       # 项目级配置（最小，仅工作区根标记）
```

**5. `cellular.toml` 最小化**：仅含 workspace root 标记 + 默认 lint 配置。**没有** cell 列表 / contract 列表 / journey 列表——这些都从 derive 派生。

#### 不引入

- ❌ Backstage / GraphQL export
- ❌ IDE LSP 插件
- ❌ Web UI graph viewer

#### 验收标准

```bash

cd mycell && cargo build                              # 通过

```

**Rust 优先验收**：

1. dylint 规则数 ≤ 8（每条 ≤ 200 LOC）
2. M0–M3 应用通过 `cellular check` 全绿
3. 故意注入违规 8 例，每例 dylint 命中 + 错误信息可读

5. 所有 derive 宏的错误诊断有 trybuild UI test

### M5：Hardening（observability + 安全）

#### 目标

把 M0–M4 接入生产级 observability + 安全栈，**端到端 PII fail-closed**。

#### Demo

`docker-compose.yml` 启动：noteshelf + searchcore + devicecore + postgres + rabbitmq + redis + jaeger + prometheus + grafana + vault。从 Jaeger UI 看一条跨 3 cell 的 trace。

#### 关键技术决策

**1. Tracing → OTLP → Jaeger**：`tracing-opentelemetry` + `opentelemetry-otlp`，cell label 从 `tracing::Span` field 读，单源。

**2. Metrics**：`metrics` crate + `metrics-exporter-prometheus`，标准 Prometheus exposition；cell label 从 tracing span field 派生。

**3. Redaction**：

```rust
pub struct Redactor { regex_set: Arc<RegexSet> }   // sealed

impl Redactor {
    pub fn redact_in_place(&self, s: &mut String) { ... }
}

// 三处 layer 共享同一 Redactor 实例：
// - tracing::layer (FormatEvent)
// - opentelemetry::SpanProcessor
// - outbox last_error sanitizer
```

dylint 规则 #1 守"Redactor 必须从 sealed constructor 构造"。

**4. Secrets**：`secrecy::Secret<T>` 类型化所有密钥，禁止 `Display` / `Debug` 默认实现；Vault 客户端走 `vaultrs`。

**5. Auth**：`jsonwebtoken` + tower layer，JWT claim 类型化（`struct AccessClaims { sub, exp, ... }`）。

**6. Rate limit / Circuit breaker**：`governor` + `failsafe`（薄自建兜底）。

#### 不引入

- ❌ K8s operator
- ❌ 多区域部署
- ❌ SOC2 audit log 完整链路（仅 demo dashboard）

#### 验收标准

```bash
docker compose up -d
curl -X POST .../notes -d '{"body":"password=secret123"}'
open http://localhost:16686                                 # Jaeger
```

**Rust 优先验收**：

1. Jaeger 一条 trace 跨 noteshelf → rabbitmq → searchcore，深度 ≥ 3
2. trace attributes 含 `cell="noteshelf"` / `cell="searchcore"`
3. 故意 `tracing::error!("password=secret123")` → Jaeger 日志显示 `password=<REDACTED>`，无 `secret123` 残留
4. `Secret<String>` 调 `Display` → **编译错误**
5. Prometheus 查询 `http_requests_total{cell="noteshelf"}` 有数据；`{cell="_runtime"}` 含 healthz / readyz

## 5. 度量与验收

### 5.1 PoC 通过判据（M0 完成后评估）

**必要条件（全部）**：

1. ✅ Cell trait + Contract trait + derive macro 三件套跑通
2. ✅ `CellCtx<L0>` / `<L1>` 一致性等级编译期不可越界
3. ✅ enum-variant 错误模型 + `IntoResponse` 对偶让 PII 在 wire 编译期不可表达
4. ✅ 单 cell 业务代码量 ≤ Go 版 60%（行数粗算）
5. ✅ derive 宏错误诊断可读（≥ 5 条 trybuild UI test）

**充分条件（≥ 2）**：

1. 🎯 编译时间增量 ≤ 60s（30 cell workspace）
2. 🎯 dylint 规则数在 M4 验证 ≤ 8
3. 🎯 团队 ≥ 2 人无障碍上手 derive 宏调试

### 5.2 失败判据（任一即停）

- ❌ async trait dyn-safe 让 cross-cell registry 必须 monomorphize 全部 cell × event 矩阵 → 编译时间爆炸
- ❌ proc macro 错误信息只能"in this macro expansion"，业务作者无法定位
- ❌ M0 工作量超预算 50%
- ❌ const generic + AtLeastLn 路线发现存在无法表达的合法用例

### 5.3 长期度量

| 指标 | 目标 |
|---|---|
| dylint 规则数 | ≤ 8 |
| 类型系统消化的 GoCell archtest | ≥ 90%（74 → ≤ 8 dylint） |
| 单 cell scaffold → 200 OK | ≤ 5 min |
| derive 宏 trybuild UI test 覆盖 | ≥ 95% |
| cargo build incremental | ≤ 60s（30 cell） |
| PII 泄漏测试用例 | 0 漏 |
