# ADR: Assembly YAML Minimal + Derivation Strategy

> Status: Accepted
> Date: 2026-05-07
> ref: docs/plans/202605011500-029-master-roadmap.md K#10
> Implementation: PR-V1-ASSEMBLY-YAML-MINIMAL（K#10，吸收 K-08(a)+K-08(c)）

## Context

原 `AssemblyMeta` 要求 6 个字段全部手填（`id`、`cells`、`build.entrypoint`、`build.binary`、`build.deployTemplate`、以及缺失的 `owner`），其中 `build.{entrypoint,binary,deployTemplate}` 三字段可从 `id` 和约定目录结构机械推导，无信息增量。`cmd/corebundle/run.go` 维护一个手工 switch，把 cell ID 字符串映射到对应的 `cell.Module` 工厂，每次新增 cell 需要人工同步编辑四处（cell.yaml、assembly.yaml、run.go switch、cmd module）。`AssemblyMeta` 缺少 `owner` 字段，导致 assembly 级别的团队归属只能靠约定而无法被 governance 工具读取。governance 层缺乏对 `maxConsistencyLevel` 的交叉校验——assembly 中 cell 的实际最高一致性等级与声明的 assembly 一致性等级可能悄悄漂移。`gocell generate assembly` 此前已能派生 `main.go` + `boundary.yaml`，但 `modules_gen.go` 仍是手工维护的空白。

## Decision

### 1. 必填字段缩到 id + cells + owner

`assembly.yaml` schema 更新后仅 `id`、`cells`（cell ID 列表）、`owner`（`OwnerMeta`，含 `team` 和 `role`）三个字段为必填。`build.entrypoint`、`build.binary`、`build.deployTemplate` 降为可选字段，schema 的 `additionalProperties: false` 配合 parser 的 `KnownFields(true)` 拒绝任何未声明字段，防止拼写错误的字段被静默忽略。

### 2. AssemblyMeta 加 Owner (OwnerMeta) + MaxConsistencyLevel (派生只读)

`Owner` 字段复用已有的 `OwnerMeta` 类型（`{team string, role string}`），在 `assembly.yaml` 中必填，直接表达 assembly 级别的团队归属。`MaxConsistencyLevel` 字段在 Go struct 中带 `yaml:"-"` tag，禁止从 YAML 文件读入——任何试图在 `assembly.yaml` 中手写 `maxConsistencyLevel:` 的行为会被 schema `additionalProperties: false` 和 parser `KnownFields(true)` 双层拦截，无需额外的 `yaml.Node` 二次扫描。`MaxConsistencyLevel` 仅由 parser 派生阶段写入，对所有消费方（generator/catalog/wire-gen）透明呈现为只读字段。

### 3. 派生发生在 parser.go::ParseFS 末段

派生逻辑集中在 `kernel/metadata/assembly_derive.go` 中的 `applyAssemblyDerivations` 函数，由 `ParseFS` 在成功解析并校验所有字段后在末段调用。`applyAssemblyDerivations` 负责：

- 从 `id` 推导 `build.entrypoint`（`cmd/{id}/main.go`）和 `build.binary`（`{id}`）（当字段为零值时）
- 将 `build.deployTemplate` 缺省为 `"k8s"`（当字段为零值时）
- 遍历 `cells` 列表，从已解析的 cell metadata 中取各 cell 的 `ConsistencyLevel`，计算最大值写入 `MaxConsistencyLevel`

governance `Validator` 在 `applyAssemblyDerivations` 完成后执行，仅做断言（read-only assert），不修改任何字段，确保 derive 与 governance 职责分离。

### 4. cmd/{id}/modules_gen.go 由 gocell generate assembly 派生

`gocell generate assembly` 命令在生成 `main.go` 和 `boundary.yaml` 的同时，额外产出 `cmd/{id}/modules_gen.go`。生成内容为：

```go
// Code generated by gocell generate assembly. DO NOT EDIT.

package main

func generatedCellModules() []CellModule {
    return []CellModule{
        {ID: "accesscore", Module: accesscore.Module{}},
        // ...
    }
}
```

编译期通过 import 路径和零值 struct 字面量化，将 cell ID 与 `cell.Module` 工厂绑定。`cmd/corebundle/run.go` 中的手工 switch 删除，改为调用 `generatedCellModules()` 并在启动时调用 `assertModuleIDsMatch` 做 fail-fast 校验，防止生成文件与实际 cell 列表漂移。

drift gate 三件套：
1. `gocell generate assembly` — 生成或更新 `modules_gen.go`
2. `gocell verify codegen-assembly` — 在 CI 中验证已提交的 `modules_gen.go` 与当前 `assembly.yaml` 一致
3. `hack/verify-codegen-assembly.sh` — 封装以上 verify 命令，作为 `hack/make-rules/verify.sh` 的一个 gate

### 5. deployTemplate enum {k8s, compose, binary} + default "k8s"

`build.deployTemplate` 的合法值限定为 `{k8s, compose, binary}` 三元枚举，在 schema 层由 `enum` 字段拒绝非法值。派生阶段将零值字段默认填充为 `"k8s"`，与 GoCell 主部署目标对齐。

### 6. governance TOPO-09 单向断言

新增 governance 规则 `TOPO-09`，在所有 cell metadata 解析完成后，断言 `assembly.MaxConsistencyLevel == max(cells[*].ConsistencyLevel)`。该规则是 read-only assert（单向断言），不修改任何字段。其目的是在 `gocell validate` 时拦截 derive 计算与实际 cell 一致性等级的潜在漂移，防止 assembly 声明的最高一致性等级与实际 cell 列表不符的情况在 CI 之外扩散。

### 7. owner 不从 cells 派生

`assembly.owner` 必须由人工明确填写，不从 cells 的 owner 字段自动取首个 cell 的 owner。原因：GoCell 主仓 `assemblies/corebundle` 包含 accesscore / auditcore / configcore 三个归属不同团队的 cell，自动取首个 cell owner 属于隐式优先级选择，会在多 owner 场景下埋下错误的归属声明，且无法被静态工具检测。强制人工填写 `owner` 要求开发者在 assembly 层面做显式的 ownership 决策。

## Rationale

- ref: kubernetes/apimachinery `pkg/apis/meta/v1/types.go` `ObjectMeta`——K8s 最小必填策略（`Name`、`Namespace` 必填，server-derived 字段如 `CreationTimestamp`、`UID` 带 `yaml:"-"` 或 `omitempty`）。GoCell `cells` 字段类比 `ObjectMeta.Name`：包含不可推导的业务语义（哪些 cell 属于这个 assembly），不可派生。
- ref: zeromicro/go-zero `tools/goctl/api/gogen/genmain.go` `genMain`——嵌入模板 + 强类型 context 的代码生成范式；与现有 `kernel/assembly/generator.go::GenerateEntrypoint` 同模式，`modules_gen.go` 作为第三个产物自然扩展。

## Consequences

**正面影响：**

- 新增 cell 的操作从 4 处编辑（cell.yaml / assembly.yaml / run.go switch / cmd module import）缩减为 3 处（cell.yaml / cmd module import / assembly.yaml cells 列表）+ 执行一次 `gocell generate assembly` codegen
- Governance 多了一道 max consistency level 安全网（TOPO-09），assembly 声明与实际 cell 组合漂移在 CI 阶段即可被拦截
- `assemblies/corebundle/assembly.yaml` 从 9 行（含整个 build 块）简化到 7 行（仅 id + cells + owner）

**负面影响：**

- `modules_gen.go` 进入 git 增加 review noise（与 `cell_gen.go` / `contract_gen.go` 既有代码生成惯例一致，drift gate 三件套锁定漂移风险）

### Negative

- 一致性等级（L0-L4）原本被两处持有：`kernel/cell.Level` 类型 + `levelStrings`
  数组（运行期类型）与 `kernel/metadata/assembly_derive.go::consistencyOrder` map
  （解析期排序）。`kernel/cell` 已 import `kernel/metadata`，反向 import 会形成
  cycle。本 PR 通过新建零依赖子包 `kernel/cell/levelrank`（仅 `Levels` 数组 +
  `Rank()` + `At()`）打破对称：`kernel/cell.Level.String/ParseLevel` 与
  `kernel/metadata.computeMaxConsistencyLevel` 都向下依赖 levelrank，单源治理。
  对标 etcd `raftpb`、CockroachDB `storage/enginepb` 子包隔离模式。L0-L4 taxonomy
  由 GoCell 宪法固化（sync/async × local/cross-cell/device），不预期扩展；本
  refactor 的目标是消除现有重复，不是为 taxonomy 扩张做准备。

## Alternatives Considered

- **runtime `init()` 注册**：放弃——违反 Go 风格（隐式副作用），引入启动顺序耦合，且不可被静态分析工具追踪。
- **`assembly.owner` 取首个 cell.owner**：放弃——多 owner 场景下隐式选择产生错误归属声明，且该错误无法被静态工具检测（见 Decision 7）。
- **派生放 governance 阶段**：放弃——generator/catalog/wire-gen 在 governance 之前消费 `AssemblyMeta`，若 `build.*` 字段在 governance 阶段才填入，对这些消费方不透明，会引发读到零值的 bug。Validator 应保持 read-only。
- **`maxConsistencyLevel` 允许 yaml override**：放弃——允许 override 会导致 governance 偏低绕过风险：开发者可以在 `assembly.yaml` 写入比实际 cell 最高等级更低的值，TOPO-09 的安全网失效。
- **examples/* 同步落 `assembly.yaml` 简化**：本 PR 不涵盖——仅 corebundle 验证派生流程；examples/ 等待 K#09 SCAFFOLD-ONE-CMD 配套时一并处理。
- **`yaml.Node` 二次扫描拒 `maxConsistencyLevel` key**：放弃——schema `additionalProperties: false` 加 parser `KnownFields(true)` 已构成双层守护，引入第三层检查增加复杂度但不增加安全性。

## Schema 约束单源

`kernel/metadata/contract_constraints.go` 中的 const（`AssemblyIDPattern`、
`GoStructNamePattern`、`DeployTemplateEnum`）是 schema 类语法约束的
authoritative Go-side source。`kernel/metadata/schemas/*.json` 文件保留字面
量供 IDE / yaml-language-server 消费，由 `kernel/metadata/schemas/
schema_const_consistency_test.go` 验证字面量与 const 字节级一致。

Runtime 单一守门员 = governance；schema-derived 规则在 default validate
路径无条件执行，与 schema-aware tooling 保持一致：

- 解析期 parser **不做值校验**，只信 schema `additionalProperties: false` +
  `KnownFields(true)` 拒未知字段；
- governance FMT-A1（assembly id 模式）默认开启，镜像 schema id.pattern；
- governance FMT-30（deployTemplate enum）默认开启，镜像 schema enum；
- typed-identifier boundary 类型（`metadata.GoIdentifier`）在构造时引用
  `goStructNameRe`。

`--strict` 模式仅承载与 schema 无关的风格性偏好（FMT-16 / FMT-C1 / FMT-17
等 kebab/allowedFiles 限制），schema 已声明的约束不再走 strict 区分。

加新约束步骤：(1) `contract_constraints.go` 加 const → (2) 同步 schema 文件
→ (3) governance 或 typed-identifier 引用 const → (4) consistency test 加比对项。

R-meta 同根 PR #403 R0：本 PR 仅做 schema 字段切面的最小封堵；通用治理
框架（`kernel/governance/invariants.go::Registry` + 四件套清单）留给
PR #403 段 2 独立 PR 落地。详见 `docs/reviews/202605070218-pr404-second-wave-review.md`。

## 派生物 typed boundary

进入 codegen template 的字段必须是 typed value，raw string 不得直接拼接进
Go 源代码模板。`kernel/metadata/identifier.GoIdentifier` 是该 boundary 的
首批实现：

- 私有 `value` 字段，仅 `NewGoIdentifier` / `MustNewGoIdentifier` /
  `UnmarshalYAML` 三条构造路径，全部经 `goStructNameRe` 校验；
- `CellMeta.GoStructName` 字段类型已迁移为 `GoIdentifier`；
- 静态保证：Go 类型系统 + 7 处 call site 的 `.IsZero()` / `.String()` 方法
  调用即拒回 raw string；任何"改回 string"的回归改动都会让所有 consumer
  编译失败，无需独立 archtest。

未来 codegen 输入字段（如新增 `cell.yaml` field 进入 template）必须采用
typed boundary，不得绕过 raw string 拼接。codegen 模板侧用 `metadata.
MustNewGoIdentifier(...)` 嵌入字面量（C-class init panic 失败模式，对应
error-handling.md 中 panic taxonomy 的初始化误配置分类）。

## archtest 守卫

- **ASSEMBLY-MODULES-GEN-01**：`cmd/*/modules_gen.go` 必须包含 `DO NOT EDIT` marker、`generatedCellModules` 函数声明、`package main` 声明。确保生成文件不被手工改写为非标准形态。
- **ASSEMBLY-MODULES-SWITCH-FORBIDDEN-02**：`cmd/*/run.go` AST 级别禁止以 cell ID 字符串字面量为 `case` 的 switch 语句。确保手工 cell→Module 映射 switch 不复出现。
- **ASSEMBLY-MAXCONSISTENCY-DERIVED-03**：`AssemblyMeta.MaxConsistencyLevel` 字段的 yaml struct tag 必须为 `"-"`。确保该字段不可被 YAML 文件写入，派生只读语义不被绕过。
- **TestSchemaConstantsMatchSchemaLiterals**（`kernel/metadata/schemas/`）：JSON schema pattern/enum 字面量必须与 `contract_constraints.go` const 字节级一致。
- **TestAssemblySpecCoversAssemblyMeta**（`runtime/devtools/catalog/`）：`metadata.AssemblyMeta` 所有 yaml 字段必须映射到 `catalog.AssemblySpec` 或在 `catalogExcludedAssemblyFields` 中显式排除（含 reason）。
