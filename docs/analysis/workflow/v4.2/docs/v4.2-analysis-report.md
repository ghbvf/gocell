> 分析日期: 2026-04-04
> 事实依据: `docs/architecture/metadata-templates-v2.md` 为元数据字段最新规范

## Context

**分析范围**:
- `docs/analysis/workflow/v4.2/` (agents, skills, rules, workflow-detailed.md)
- `docs/architecture/metadata-templates-v2.md` (V2 事实依据)
- `CLAUDE.md` + `docs/architecture/` 下关联文档

## P0 发现 (阻塞级)

### 集群 1: V1→V2 元数据迁移缺失 (4 条)

> **根因**: metadata-templates-v2.md 作为新规范写入,但下游 7 个消费文件从未更新。V2 仅作为文档存在,对工作流无实际影响力。

#### P0-01: go-standards.md 使用 V1 废弃字段名
- **角色**: 文档工程师 + Kernel Guardian
- **位置**: `v4.2/.claude/rules/go-standards.md:118-136`
- **证据**:
  - go-standards.md: `cellId`, `ownedSlices`, `authoritativeData`, `sliceId`, `journeys`(slice)
  - V2 废弃列表(line 31-38): 明确禁止 `cellId`, `sliceId`, `ownedSlices`, `authoritativeData`
- **影响**: 该规则通过 frontmatter paths 自动加载到所有 Go 文件,是 V1 字段传播的最大载体
- **修复**: 替换为 V2 字段: cell.yaml → `id, type, consistencyLevel, owner{team,role}, schema.primary, verify.smoke`; slice.yaml → `id, belongsToCell, consistencyLevel, owner{team,role}, callsContracts, publishes, consumes, verify.unit, verify.contract, allowedFiles`

#### P0-02: kernel-guardian.md S2 约束清单使用 V1 字段
- **角色**: Kernel Guardian + Harness 专家
- **位置**: `v4.2/.claude/agents/kernel-guardian.md:56`
- **证据**: `cell.yaml 必须含 cellId/type/consistencyLevel/owner/ownedSlices/contracts/verify`
- **影响**: Kernel Guardian 在 S2 产出的 kernel-constraints.md 将以 V1 字段为基准,S4/S6/S8 全链路使用错误字段
- **修复**: 同 P0-01,更新为 V2 字段

#### P0-03: 根 CLAUDE.md 使用 V1 字段
- **角色**: 文档工程师
- **位置**: `CLAUDE.md:36-37`
- **证据**: `cellId / type / consistencyLevel / owner / ownedSlices / authoritativeData / contracts / verify`
- **影响**: 每个 Claude Code 会话加载此文件,V1 字段成为默认认知
- **修复**: 更新为 V2 字段,引用 metadata-templates-v2.md

#### P0-04: V2 新增必填字段在工作流中完全缺失
- **角色**: Kernel Guardian + 架构师
- **位置**: 整个 v4.2 目录
- **证据**: grep `schema.primary`, `verify.smoke`, `verify.unit`, `verify.contract`, `callsContracts`, `publishes`, `consumes` 在 v4.2 目录返回零结果
- **影响**: 这些字段是 V2 的 20 条验证规则的基础(规则 5-8, 14-16),完全无执行路径

#### P0-08: 所有 SKILL.md 的 phase-gate-check.sh 调用缺少 `--branch` 参数
- **角色**: Harness 专家 + DevOps/QA
- **位置**: 所有 9 个 stage SKILL.md 文件
- **证据**:

- **证据**: `ls scripts/bash/` 仅有 `phase-gate-check.sh`
- **影响**: S1 第一步即失败,无法创建 feature branch
- **修复**: 创建脚本或替换为等效 git 命令

#### P0-10: V2 的 20 条验证规则仅 2 条有执行路径
- **角色**: Kernel Guardian + 架构师

- **证据**: 20 条规则中: 规则 1 和 14 有部分覆盖,规则 19 提及但未强制执行,其余 17 条零覆盖。包括:

  - 规则 6-8: publishes/callsContracts/consumes 验证 — 零引用
  - 规则 9: actor registry 验证 — actors.yaml 在 v4.2 零引用
  - 规则 15: contract kind + consistencyLevel 组合约束 — 零引用
- **修复**: 将关键规则纳入 kernel-guardian S2 约束清单和 S4 任务审查

## 根因分析

```
metadata-templates-v2.md 写入
        │
        ├─ CLAUDE.md ──────────── 未更新 (V1 字段)
        ├─ go-standards.md ────── 未更新 (V1 字段 + 引用 V1 文档)
        ├─ kernel-guardian.md ─── 未更新 (V1 字段 + 维度不一致)
        ├─ product-manager.md ── 未更新 (维度不一致 + 引用 catalog.yaml)
        ├─ metadata-schemas.md ── 未更新 (V1 字段 + V1/V2 嵌合)
        ├─ glossary.md ────────── 未更新 (V1 字段定义)
        └─ master-plan.md ─────── 未更新 (V1 字段示例)
```

同时,v4.2 workflow 从 v4.1 演进时:
- 维度定义在 workflow-detailed.md 和 stage-8-close/SKILL.md 中更新
- 但 agent 定义文件(kernel-guardian.md, product-manager.md)未同步
- SKILL.md quick-reference 表格未从 phase-gates.yaml 重新生成
- stage-7-qa SKILL.md 遗漏填充
